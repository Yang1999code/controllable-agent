### 核心流程

```
用户对话结束 → LOOP_END 携带 AgentResult
    → MemoryExtractor.check_and_extract()
        → TaskDetector.detect() 启发式判断任务完成
            → 完成(confidence>=0.6) → 调用 LLM 提取 digest
                → FactStore.create_digest() 写入
                → DomainIndex.update_index_for() 更新索引
                → 同主题 digest >= 5? → wiki 合并
            → 未完成 → 跳过
```

### 关键设计决策

1. **固定 system prompt**：`_DIGEST_SYSTEM_PROMPT` 和 `_WIKI_MERGE_SYSTEM_PROMPT` 永不变，KV Cache 友好
2. **JSON 输出格式**：LLM 返回结构化 JSON，三种容错解析（纯 JSON / ```json``` / 混合文本）
3. **记忆提取不阻塞主循环**：提取在 LOOP_END 之后执行，异常不致命
4. **TaskDetector 纯规则**：Phase 2 不调 LLM，Phase 3 可替换
5. **Deduplicator 4 种判断**：new / duplicate / conflict / supplement，conflict+supplement 时给出 merged_body
