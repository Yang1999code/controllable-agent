# 部署文档

本文档描述如何部署 Controllable Agent 到不同环境。

## 快速开始

### 1. 本地开发环境

```bash
# 克隆仓库
git clone https://github.com/Yang1999code/controllable-agent.git
cd controllable-agent

# 创建 conda 环境
conda create -n controllable-agent python=3.12
conda activate controllable-agent

# 安装依赖
pip install -e ".[all]"

# 配置 API Key
cp .env.example .env
# 编辑 .env 填入你的 API Key

# 启动 CLI
python -m app.cli

# 或启动 REST API 服务器
python -m app.server
# 访问 http://localhost:8000/docs 查看 API 文档
```

### 2. Docker 部署

```bash
# 构建镜像
docker build -t controllable-agent .

# 运行 CLI
docker run -it --rm \
  -v $(pwd)/app/config/agent.yaml:/app/app/config/agent.yaml \
  -e DEEPSEEK_API_KEY=sk-your-key \
  controllable-agent

# 或使用 docker-compose
docker-compose up -d

# 访问 REST API
curl http://localhost:8000/health

# 查看日志
docker-compose logs -f api
```

### 3. Kubernetes 部署

```yaml
# k8s/deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: controllable-agent
spec:
  replicas: 3
  selector:
    matchLabels:
      app: controllable-agent
  template:
    metadata:
      labels:
        app: controllable-agent
    spec:
      containers:
      - name: agent
        image: controllable-agent:latest
        ports:
        - containerPort: 8000
        env:
        - name: DEEPSEEK_API_KEY
          valueFrom:
            secretKeyRef:
              name: agent-secrets
              key: deepseek-api-key
        livenessProbe:
          httpGet:
            path: /health
            port: 8000
          initialDelaySeconds: 30
          periodSeconds: 10
---
apiVersion: v1
kind: Service
metadata:
  name: controllable-agent
spec:
  selector:
    app: controllable-agent
  ports:
  - protocol: TCP
    port: 80
    targetPort: 8000
  type: LoadBalancer
```

## REST API 使用示例

### 健康检查
```bash
curl http://localhost:8000/health
```

### 获取状态
```bash
curl http://localhost:8000/status
```

### 单次对话
```bash
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "你好，请帮我写一个 Python 函数"}'
```

### 异步任务
```bash
# 提交任务
TASK_ID=$(curl -s -X POST http://localhost:8000/run \
  -H "Content-Type: application/json" \
  -d '{"task": "写一个排序算法", "multi_agent": false}' \
  | jq -r '.task_id')

# 查询状态
curl http://localhost:8000/task/$TASK_ID
```

### WebSocket 连接
```python
import asyncio
import websockets

async def connect():
    async with websockets.connect("ws://localhost:8000/ws") as ws:
        await ws.send_json({"type": "message", "content": "你好"})
        async for message in ws:
            print(message)

asyncio.run(connect())
```

## 环境变量

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `DEEPSEEK_API_KEY` | DeepSeek API Key | - |
| `DASHSCOPE_API_KEY` | 通义千问 API Key | - |
| `OPENAI_API_KEY` | OpenAI API Key | - |
| `ANTHROPIC_API_KEY` | Anthropic API Key | - |
| `MY_AGENT_MAX_CONTEXT_TOKENS` | 最大上下文 token 数 | 128000 |

## 性能调优

### 生产环境建议

1. **使用 Gunicorn + Uvicorn**
```bash
pip install gunicorn
gunicorn app.server:app -w 4 -k uvicorn.workers.UvicornWorker
```

2. **增加 Worker 数量**
```bash
uvicorn app.server:app --host 0.0.0.0 --port 8000 --workers 4
```

3. **配置反向代理 (Nginx)**
```nginx
server {
    listen 80;
    server_name agent.example.com;

    location / {
        proxy_pass http://localhost:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 故障排查

### 常见问题

1. **API Key 无效**
   - 检查 `.env` 文件是否存在
   - 确认环境变量已加载

2. **Docker 容器启动失败**
   - 检查端口是否被占用
   - 查看日志 `docker-compose logs`

3. **测试失败**
   - 运行 `pytest tests/ -v` 查看详细错误
   - 确保 Python 版本 >= 3.12
