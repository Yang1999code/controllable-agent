## 一、架构全景

### 1.1 三层架构（Pi Agent 模式，单向向下依赖）

```
ai/          ← 零依赖层：纯类型 + 抽象接口。不 import 任何项目内模块
agent/       ← 核心逻辑层：20 个接口实现。只依赖 ai/，不依赖 app/
app/         ← 具体实现层：CLI + 内置工具 + 提供商 + 配置文件。依赖 ai/ + agent/
```

**铁律**：
- `ai/` 禁止 import `agent/` 或 `app/`
- `agent/` 禁止 import `app/`
- `app/` 可 import 任何层
- 每个接口先写 ABC，再写实现。预留接口只需要 ABC 签名（`...` 或 `pass`）

### 3.1 ai/types.py — 核心类型定义

**职责**：定义全项目共享的 dataclass / Protocol。零依赖，最先写。

**参考**：
- Pi Agent `AgentTool<T>` 泛型 + `AgentMessage` 联合类型
  - 路径：`controable code\pi-mono-main\pi-mono-main\packages\agent\src\types.ts`
- CCB `Tool<Input, Output, Progress>` 泛型
  - 路径：`controable code\claude-code-main\claude-code-main\src\types\message.ts`

**实现内容**：

```python
from dataclasses import dataclass, field
from typing import Any, Callable, Literal, Protocol, AsyncIterator, runtime_checkable
from enum import Enum


# ── Message ───────────────────────────────────────────
@dataclass
class Message:
    """统一消息类型。role 区分来源。"""
    role: Literal["system", "user", "assistant", "tool"]
    content: str
    tool_call_id: str | None = None       # tool 角色时使用
    tool_name: str | None = None          # tool 角色时使用
    metadata: dict = field(default_factory=dict)  # 预留：token用量等


# ── Tool ──────────────────────────────────────────────
@dataclass
class ToolParameter:
    """工具参数定义（JSON Schema 子集）。"""
    name: str
    type: str = "string"
    description: str = ""
    required: bool = False
    default: Any = None
    enum: list[str] | None = None

@dataclass
class ToolDefinition:
    """工具的静态描述——发给 LLM 的 JSON Schema。"""
    name: str
    description: str
    parameters: list[ToolParameter] = field(default_factory=list)

@dataclass
class ToolResult:
    """工具执行结果。"""
    tool_name: str
    success: bool
    content: str                          # 文本结果
    error: str | None = None
    truncated: bool = False               # 结果是否被截断
    file_path: str | None = None          # 截断时写入的文件路径

@runtime_checkable
class ITool(Protocol):
    """工具接口（Protocol，鸭子类型，不需要显式继承）。

    参考：GenericAgent BaseHandler.dispatch() 模式
    路径：controable code\GenericAgent-main\GenericAgent-main\agent_loop.py:18-28
    """
    definition: ToolDefinition            # 静态描述
    is_concurrency_safe: bool             # 是否可以并行执行

    async def execute(self, args: dict, context: "Context") -> ToolResult:
        """执行工具。args 由 LLM 生成，需要校验。"""
        ...


# ── Context ───────────────────────────────────────────
@dataclass
class Context:
    """Agent 上下文，在循环中传递和修改。

    参考：Pi Agent agent-loop.ts AgentContext
    路径：controable code\pi-mono-main\pi-mono-main\packages\agent\src\agent-loop.ts:15-24
    """
    system_prompt: str = ""
    messages: list[Message] = field(default_factory=list)
    tools: dict[str, ITool] = field(default_factory=dict)
    metadata: dict = field(default_factory=dict)  # session_id / project_path 等


# ── Agent Event ───────────────────────────────────────
class AgentEventType(Enum):
    # Phase 1（需求2）
    TURN_START = "turn_start"
    TURN_END = "turn_end"
    TOOL_START = "tool_start"
    TOOL_END = "tool_end"
    LOOP_START = "loop_start"
    LOOP_END = "loop_end"
    STEER_INJECT = "steer_inject"
    ERROR = "error"
    # Phase 2（需求3）
    SESSION_START = "session_start"
    SESSION_END = "session_end"
    LLM_CALL = "llm_call"
    PLUGIN_LOADED = "plugin_loaded"
    PLUGIN_UNLOADED = "plugin_unloaded"
    # Phase 3（需求4）
    SUBAGENT_START = "subagent_start"
    SUBAGENT_STOP = "subagent_stop"

@dataclass
class AgentEvent:
    type: AgentEventType
    data: dict = field(default_factory=dict)
    timestamp: float = 0.0
```

**关键决策**：
- `ITool` 用 `Protocol` 而非 ABC——允许函数/闭包伪装成工具，无需显式继承
- `Message` 单一类型 + role 字段——避免联合类型的复杂 match 逻辑
- `Context` 是可变 dataclass——循环中直接 append message，无需 immutable 模式

**依赖**：无（零依赖）

**验收**：所有类型可 import，无循环引用

**职责**：统一的 LLM 调用抽象。V1 只做 OpenAI 兼容 + Anthropic 两个。

**参考**：

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass
from typing import AsyncIterator
from .types import Message, ToolDefinition


@dataclass
class LLMEvent:
    """流式事件。type 区分内容类型。"""
    type: Literal["text_delta", "tool_call", "tool_call_args", "done", "error"]
    content: str = ""
    tool_name: str = ""
    tool_id: str = ""
    usage: dict = field(default_factory=dict)  # {input_tokens, output_tokens}

    - OpenAICompatibleProvider（覆盖 OpenAI/DeepSeek/通义千问/智谱等）
    - AnthropicProvider（Anthropic 原生 Messages API）
    """

    @abstractmethod
    async def stream(
        self,
        messages: list[Message],
        tools: list[ToolDefinition],
        system_prompt: str = "",
        max_tokens: int = 4096,
        temperature: float = 0.0,

        参考 CCB query.ts 的 stream 处理逻辑。
        """
        ...

    @abstractmethod
    async def chat(
        self,
        messages: list[Message],
        tools: list[ToolDefinition],
        system_prompt: str = "",
        max_tokens: int = 4096,
    ) -> list[LLMEvent]:
        """非流式调用 LLM（用于子Agent等场景）。"""
        ...

    @abstractmethod
    def count_tokens(self, text: str) -> int:
        """粗略 token 计数（用于预算控制，不需要精确）。"""
        ...


class OpenAICompatibleProvider(IModelProvider):
    """OpenAI 兼容协议实现。

    初始化参数：
    - base_url: API 端点（默认 https://api.openai.com/v1）

    - model: 模型名（默认 gpt-4o）

    实现要点：
    1. HTTP 客户端用 httpx（支持 HTTP/2 + 连接池）
    2. 工具定义转为 OpenAI function calling 格式
    3. 流式响应用 SSE 解析（httpx.stream + iter_lines）
    4. 自动重试（429/5xx，指数退避，最多3次）
    """
    pass  # V1 实现


class AnthropicProvider(IModelProvider):
    """Anthropic Messages API 实现。

    初始化参数：
    - api_key: 从环境变量 ANTHROPIC_API_KEY 读取
    - model: 模型名（默认 claude-sonnet-4-6）

    实现要点：
    1. 使用 anthropic SDK 的 async client
    2. 工具定义转为 Anthropic tool_use 格式
    3. 流式响应用 async for chunk
    """
    pass  # V1 实现
```

**关键决策**：
- V1 不做凭证池（V2 预留），API key 从环境变量 `{PROVIDER}_API_KEY` 读取
- `stream()` 返回 `AsyncIterator[LLMEvent]` 而非一次性返回——内存友好 + 支持增量渲染
- token 计数用 `len(text) // 3.5` 近似（英文）或 `len(text) // 2` 近似（中文），不做精确计数
- 自动重试：429 等 `Retry-After` 秒数，5xx 指数退避（1s→2s→4s）

**常见陷阱**：
- OpenAI 兼容协议各厂商实现不一致（如 DeepSeek 的 function calling 格式有差异）→ 加 adapter 层
- Anthropic 的 tool_use content block 需要特殊处理（非标准 message content）
- 流式中断时未正确关闭 httpx 连接 → 用 `async with` 或 try/finally

### 3.3 agent/step_outcome.py — 控制流原语

**职责**：循环中每一步的返回控制——继续 / 退出 / 注入新 prompt。避免复杂的枚举状态机。

**参考**：
- GenericAgent `StepOutcome` dataclass（极简）
  - 路径：`controable code\GenericAgent-main\GenericAgent-main\agent_loop.py:5-7`
- Pi Agent tool result → next_prompt 逻辑
  - 路径：`controable code\pi-mono-main\pi-mono-main\packages\agent\src\agent-loop.ts`

**实现内容**：

```python
from dataclasses import dataclass, field
from typing import Any


@dataclass
class StepOutcome:
    """工具执行后的控制原语。

    参考 GenericAgent agent_loop.py:5-7 的三字段设计：
    - data: 工具返回数据
    - next_prompt: 下一轮的 user prompt（None = 无后续）
    - should_exit: 是否终止循环

    扩展版增加 steer 和 error 字段。
    """
    data: Any = None
    next_prompt: str | None = None       # None → 让 LLM 决定下一步
    should_exit: bool = False            # True → 终止循环
    exit_reason: str = ""                # "completed" / "max_turns" / "user_abort"
    steer_message: str | None = None     # 注入 steer 队列的消息
    error: str | None = None             # 工具执行错误

    # 工厂方法
    @classmethod
    def done(cls, data: Any = None) -> "StepOutcome":
        """任务正常完成。"""
        return cls(data=data, should_exit=True, exit_reason="completed")

    @classmethod
    def continue_(cls, next_prompt: str) -> "StepOutcome":
        """继续执行，指定下一轮 prompt。"""
        return cls(next_prompt=next_prompt)

    @classmethod
    def steer(cls, message: str) -> "StepOutcome":
        """注入一条 steer 消息。"""
        return cls(steer_message=message)
```

**设计决策**：
- next_prompt=None 且 should_exit=False → LLM 决定下一步（最常见）
- should_exit=True → 无论 next_prompt 是什么都退出循环
- steer_message 独立于 next_prompt —— steer 是"外部注入的指令"，next_prompt 是"工具要求的继续"

**依赖**：无（纯 dataclass）

```python
import asyncio
import hashlib
from pathlib import Path
from dataclasses import dataclass, field
from ai.types import ITool, ToolDefinition, ToolResult, Context


@dataclass
class ToolRegistry:
    """工具注册表。

    核心功能：
    1. 注册/注销工具
    2. 并发安全分组执行
    3. 结果预算裁剪（超限写磁盘）
    """
    tools: dict[str, ITool] = field(default_factory=dict)
    max_result_chars: int = 50000          # 单工具结果字符上限

    # ── 注册 ──

    def register(self, tool: ITool) -> None:
        """注册工具。同名工具后者覆盖前者。"""
        if not tool.definition.name:
            raise ValueError("Tool name cannot be empty")
        self.tools[tool.definition.name] = tool

    def unregister(self, name: str) -> None:
        self.tools.pop(name, None)

    def get_definitions(self) -> list[ToolDefinition]:
        """获取所有工具的 JSON Schema 列表（发给 LLM）。"""
        return [t.definition for t in self.tools.values()]

    # ── 执行调度 ──

    async def execute_many(
        self, tool_calls: list[dict], context: Context
    ) -> list[ToolResult]:
        """执行多个工具调用。安全的并行，不安全的串行。

        参考 CCB tool execution 逻辑：is_concurrency_safe 声明 + asyncio.gather。
        """
        safe_calls, unsafe_calls = [], []
        for tc in tool_calls:
            tool = self.tools.get(tc["tool_name"])
            if tool and tool.is_concurrency_safe:
                safe_calls.append((tool, tc))
            elif tool:
                unsafe_calls.append((tool, tc))

        results = []

        # 安全的并行执行
        if safe_calls:
            safe_results = await asyncio.gather(*[
                self._execute_one(tool, tc["args"], context)
                for tool, tc in safe_calls
            ])
            results.extend(safe_results)

        # 不安全的串行执行
        for tool, tc in unsafe_calls:
            result = await self._execute_one(tool, tc["args"], context)
            results.append(result)

        return results

    async def _execute_one(
        self, tool: ITool, args: dict, context: Context
    ) -> ToolResult:
        """执行单个工具 + 结果预算裁剪。"""
        try:
            result = await tool.execute(args, context)
            return self._apply_budget(result)
        except Exception as e:
            return ToolResult(
                tool_name=tool.definition.name,
                success=False,
                content="",
                error=str(e),
            )

    def _apply_budget(self, result: ToolResult) -> ToolResult:
        """结果超 → 写磁盘，替换为路径引用。

        参考 CCB applyToolResultBudget 逻辑：
        - 超过 max_result_chars 的结果写入临时文件
        - 原 result.content 替换为文件路径引用
        """
        if len(result.content) <= self.max_result_chars:
            return result

        # 写入 .agent-memory/{project}/tool_results/{hash}.txt
        result_hash = hashlib.sha256(result.content.encode()).hexdigest()[:16]
        result_dir = Path(".agent-memory/tool_results")
        result_dir.mkdir(parents=True, exist_ok=True)
        file_path = result_dir / f"{result_hash}.txt"
        file_path.write_text(result.content, encoding="utf-8")

        result.content = (
            f"[结果过长，已写入文件] 路径: {file_path}\n"
            f"前 1000 字符预览:\n{result.content[:1000]}..."
        )
        result.truncated = True
        result.file_path = str(file_path)
        return result

    # ── 工具校验 ──

    @staticmethod
    def validate_args(tool: ITool, args: dict) -> list[str]:
        """校验工具参数。返回错误列表，空列表=合法。

        参考 Pi Agent validateToolArguments()。
        路径：controable code\pi-mono-main\pi-mono-main\packages\ai\src\
        """
        errors = []
        for param in tool.definition.parameters:
            if param.required and param.name not in args:
                errors.append(f"缺少必需参数: {param.name}")
            if param.name in args and param.enum:
                if args[param.name] not in param.enum:
                    errors.append(f"参数 {param.name} 不在允许范围: {param.enum}")
        return errors
```

**关键决策**：
- `is_concurrency_safe` 布尔声明（参考 CCB）—— V1 不做三阶段 prepare/execute/finalize（Pi Agent 模式，V2预留）
- 工具执行异常不向上抛——全部捕获为 ToolResult(success=False)，保证循环不中断
- 结果预算默认 50000 字符——足够大部分场景，超限写磁盘避免污染 LLM 上下文
- 工具 args 由 LLM 生成，必须校验（至少检查 required 参数）

- BashTool 永远标记 `is_concurrency_safe=False`（多个 shell 并行会冲突）
- 文件读写工具在同一文件上的并发操作——需要文件锁
- LLM 生成的 args 可能包含未声明参数→忽略多余参数而非报错

```python
import asyncio
import logging
from typing import Any, Protocol, Callable
from dataclasses import dataclass, field
from ai.types import AgentEvent, AgentEventType

logger = logging.getLogger(__name__)


class IHook(ABC):
    """Hook 系统抽象接口。

    定义事件驱动的插件扩展协议。每个事件点持有一个 Handler 列表，
    按优先级排序执行，异常隔离保证单个 handler 失败不影响其余。
    """

    @abstractmethod
    def register(self, handler: "HookHandler") -> None:
        """注册 hook 处理器。"""
        ...

    @abstractmethod
    def unregister(self, name: str) -> None:
        """按名称注销 hook 处理器。"""
        ...

    @abstractmethod
    async def fire(self, event: AgentEvent, chain_result: bool = False) -> list[Any]:
        """触发事件。依次执行所有匹配的 handler。"""
        ...


@dataclass
class HookHandler:
    """单个 Hook 处理器。"""
    name: str
    event_type: AgentEventType
    callback: Callable[[AgentEvent], Any]
    priority: int = 50                       # 0-100，数字越小越先执行
    block_on_error: bool = False             # 异常是否阻断后续
    enabled: bool = True

    def __post_init__(self):
        if not 0 <= self.priority <= 100:
            raise ValueError(f"priority must be 0-100, got {self.priority}")


class HookChain(IHook):
    """Hook 事件链执行器（IHook 实现）。

    每个事件点持有一个 Handler 列表，按 priority 排序执行。
    核心原则（来自 oh-my-opencode 反面教训）：
    - 每个 handler 独立 try/except，失败记日志继续
    - chain_result=False 时短路（异常不阻断 → 默认行为）
    - chain_result=True 时异常阻断后续 handler

    参考 oh-my-opencode hooks 异常隔离模式。
    路径：controable code\oh-my-opencode-dev\oh-my-opencode-dev\src\
    """

    def __init__(self):
        self._handlers: dict[AgentEventType, list[HookHandler]] = {
            etype: [] for etype in AgentEventType
        }

    def register(self, handler: HookHandler) -> None:
        """注册 hook。"""
        self._handlers[handler.event_type].append(handler)
        self._handlers[handler.event_type].sort(key=lambda h: h.priority)

    def unregister(self, name: str) -> None:
        """按名称注销 hook。"""
        for handlers in self._handlers.values():
            handlers[:] = [h for h in handlers if h.name != name]

    async def fire(self, event: AgentEvent, chain_result: bool = False) -> list[Any]:
        """触发事件。依次执行所有匹配的 handler。

        chain_result=False: 异常不阻断，返回所有成功结果
        chain_result=True: 异常阻断后续 handler
        """
        results = []
        for handler in self._handlers.get(event.type, []):
            if not handler.enabled:
                continue
            try:
                result = handler.callback(event)
                if asyncio.iscoroutine(result):
                    result = await result
                results.append(result)
            except Exception as e:
                logger.warning(
                    f"Hook '{handler.name}' failed on {event.type.value}: {e}"
                )
                if chain_result:
                    raise
        return results


# ── V1 事件点定义 ─────────────────────────────

# 需求2 基础事件（7个，V1 实现）
V1_EVENTS = {
    AgentEventType.TURN_START,      # 每轮 LLM 调用开始
    AgentEventType.TURN_END,        # 每轮 LLM 调用结束（Phase 3: +检查点+Nudge）
    AgentEventType.TOOL_START,      # 工具执行前（可 block）
    AgentEventType.TOOL_END,        # 工具执行后（可覆盖结果）
    AgentEventType.LOOP_START,      # 外层循环开始
    AgentEventType.LOOP_END,        # 外层循环结束
    AgentEventType.ERROR,           # 全局异常
}

# 需求3 扩展事件（7个，V2/V1 混合）
V2_EVENTS = {
    # Phase 2 即实现
    AgentEventType.LLM_CALL,        # LLM 调用（before/after 合并）
    # Phase 2 插件事件
    # "plugin_loaded",
    # "plugin_unloaded",
    # Phase 3
    AgentEventType.SUBAGENT_START,  # 子Agent启动

```

- 每个 handler 独立 try/except（参考 oh-my-opencode 异常隔离）。一个 hook 坏了不能拖垮整个循环

```python
# agent/memory/store.py — 文件系统实现

class MemoryStore:
    """文件系统记忆存储。

    核心公理（来自 GenericAgent）：
    1. No Execution, No Memory — 只有工具验证过的才记
    2. 神圣不可删改 — 验证过的信息不能丢
    3. 禁止存储易变状态 — 不记临时数据
    4. 最小充分指针 — 上层只留能定位下层的最短标识
    """

    def __init__(self, base_path: str = ".agent-memory"):
        self.base_path = Path(base_path)
        self._lock = asyncio.Lock()  # 文件写并发保护

    async def write(self, path: str, content: str) -> None:
        """写入文件。自动创建父目录。"""
        async with self._lock:
            full_path = self.base_path / path
            full_path.parent.mkdir(parents=True, exist_ok=True)
            full_path.write_text(content, encoding="utf-8")

    async def read(self, path: str) -> str | None:
        """读取文件。不存在返回 None。"""
        full_path = self.base_path / path
        if full_path.exists():
            return full_path.read_text(encoding="utf-8")
        return None

    async def glob(self, pattern: str) -> list[Path]:
        """搜索文件。"""
        return list(self.base_path.glob(pattern))


# agent/memory/index.py — L0-L4 索引管理

class MemoryIndex:
    """L0-L4 记忆索引。

    参考 GenericAgent memory_management_sop.md 的层级定义：
    - L0（全局索引）≤30行，1K tokens
    - L1（导航层）场景关键词 → 存储定位
    - L2（事实库）环境事实、路径、配置
    - L3（任务经验）特定任务 SOP、避坑指南
    - L4（原始会话）对话记录归档
    """

    def __init__(self, store: MemoryStore):
        self.store = store

    @staticmethod
    def _tokenize(text: str) -> list[str]:
        """中文分词 + 英文分词统一入口。

        优先使用 jieba 分词（中文友好），jieba 不可用时回退到字符二元组。
        英文部分始终按空格和标点拆分。
        此方法为规范副本（canonical），AgentRuntime._tokenize() 须与此保持一致。
        """
        try:
            import jieba
            # jieba 分词：精确模式（默认）
            words = jieba.lcut(text.lower())
            # 过滤纯标点/空白
            import re
            return [w.strip() for w in words
                    if w.strip() and not re.match(r'^[\s\W_]+$', w)]
        except ImportError:
            pass

        # 回退：字符二元组（bigram），对中文基本可用
        clean = text.lower().strip()
        # 英文部分按空格拆分
        import re
        tokens = []
        # 提取英文单词
        eng_words = re.findall(r'[a-z0-9]+', clean)
        tokens.extend(eng_words)
        # 中文/混合部分用二元组
        chinese_chars = re.findall(r'[一-鿿]+', clean)
        for segment in chinese_chars:
            for i in range(len(segment) - 1):
                tokens.append(segment[i:i+2])
            if len(segment) == 1:
                tokens.append(segment)
        return tokens

    def search_keywords(self, query: str) -> list[str]:
        """中文友好的关键词检索。V1 不做 embedding。

        分词：用 _tokenize() 统一分词（jieba 优先，字符二元组回退）
        匹配度：按命中次数排序，返回匹配文件路径列表
        """
        keywords = self._tokenize(query)
        if not keywords:
            return []

        # 对每个文件计算命中分数
        scores: dict[str, int] = {}
        for f in self.store.glob("**/*.md"):
            try:
                content = self.store.read(str(f))
            except Exception:
                continue
            if not content:
                continue
            content_lower = content.lower()
            score = 0
            for kw in keywords:
                # 每个关键词命中计 1 分
                count = content_lower.count(kw)
                if count > 0:
                    score += count
            if score > 0:
                scores[str(f)] = score

        # 按命中次数降序排序
        return [f for f, _ in sorted(scores.items(), key=lambda x: x[1], reverse=True)]
```

**设计决策**：
- V1 纯文件系统（MD 文件 + 关键词匹配），不做 embedding
- 中文分词用 jieba（精确模式），不可用时回退到字符二元组（bigram）——零依赖风险
- `_tokenize()` 规范副本在 `agent/memory/index.py`，`AgentRuntime._tokenize()` 为使用副本
- 匹配度按命中次数降序排序，而非简单去重——提高检索精度
- 存储路径严格遵循目录树规范
- 索引规则：单文件超 100 行拆分，超 50 会话归档
- `asyncio.Lock` 保护文件写操作——避免并发写导致数据损坏
- MD 格式用于会话记忆/检查点/索引（人可读 + Agent tool 可直接读写）
- YAML 格式用于技能卡/配置（结构化数据 + Python 原生加载）

**常见陷阱**：
- 中文分词 jieba 未安装时回退到 bigram——召回率下降约 30%，建议生产环境安装 jieba

```python
        # Phase 3
        self.runtime = runtime
        self.autonomous_memory = autonomous_memory

    async def run(self, user_input: str, context: Context) -> AgentResult:
        """主入口。"""
        # 初始化
        turn_count = 0
        total_tool_calls = 0
        total_input_tokens = 0
        total_output_tokens = 0

        # 添加用户消息
        context.messages.append(Message(role="user", content=user_input))

        # 外层 followUp 循环
        await self.hooks.fire(AgentEvent(type=AgentEventType.LOOP_START))

        follow_up_queue: asyncio.Queue[str] = asyncio.Queue()
        steer_queue: asyncio.Queue[str] = asyncio.Queue()

        # 启动时把 user_input 放入 followUp 队列
        await follow_up_queue.put(user_input)

        while turn_count < self.config.max_turns:
            # 检查 steer 消息（优先级高于 followUp）
            next_message = None
            try:
                next_message = steer_queue.get_nowait()
            except asyncio.QueueEmpty:
                try:
                    next_message = follow_up_queue.get_nowait()
                except asyncio.QueueEmpty:
                    break  # 无事可做，退出

            if not next_message:
                break

            turn_count += 1
            await self.hooks.fire(AgentEvent(
                type=AgentEventType.TURN_START,
                data={"turn": turn_count, "message": next_message}
            ))

            # ── Phase 2 嵌入：动态构建 system prompt ──
            if self.prompt_builder:
                context.system_prompt = self.prompt_builder.build(context)

            # ── 内层 tool_calls 循环 ──
            tool_call_count = 0
            while tool_call_count < self.config.max_tool_calls_per_turn:
                # LLM 调用
                await self.hooks.fire(AgentEvent(
                    type=AgentEventType.LLM_CALL,
                    data={"messages": len(context.messages)}
                ))

                response_text = ""
                tool_calls = []

                async for event in self.provider.stream(
                    messages=context.messages,
                    tools=self.tools.get_definitions(),
                    system_prompt=context.system_prompt,
                ):
                    if event.type == "text_delta":
                        response_text += event.content
                    elif event.type == "tool_call":
                        tool_calls.append({
                            "tool_name": event.tool_name,
                            "tool_id": event.tool_id,
                            "args": {}
                        })
                    elif event.type == "tool_call_args":
                        # 累积 tool call 的参数 JSON
                        if tool_calls:
                            tool_calls[-1]["args_raw"] = (
                                tool_calls[-1].get("args_raw", "")
                                + event.content
                            )
                    elif event.type == "done":
                        total_input_tokens += event.usage.get("input_tokens", 0)
                        total_output_tokens += event.usage.get("output_tokens", 0)

                # 没有工具调用 → 退出内层循环，进入下一轮 followUp
                if not tool_calls:
                    # 添加 assistant message
                    context.messages.append(Message(
                        role="assistant", content=response_text
                    ))
                    break

                tool_call_count += len(tool_calls)
                total_tool_calls += len(tool_calls)

                # 执行工具
                results = await self.tools.execute_many(tool_calls, context)

                # 注入工具结果到上下文
                for tc, result in zip(tool_calls, results):
                    context.messages.append(Message(
                        role="tool",
                        content=result.content,
                        tool_call_id=tc.get("tool_id", ""),
                        tool_name=result.tool_name,
                    ))

                # ★ Phase 3：检查子Agent收件箱（Agent间通信）
                if self.runtime:
                    inbox_msg = self.runtime.check_inbox(
                        context.metadata.get("agent_id", "main")
                    )
                    if inbox_msg:
                        context.messages.append(Message(
                            role="user",
                            content=f"[子Agent消息] {inbox_msg}"
                        ))

                # Phase 3 嵌入：委托由 DelegateTaskTool.execute() 驱动
                # tool_calls 中包含 delegate_task → execute() 内调用 runtime.spawn()
                # → 阻塞等待子Agent完成 → 结果作为 ToolResult 注入 context
                # 无需额外检查——工具执行机制自然完成委托路由

            # ── turn_end Hook + 自进化嵌入（Phase 3）──
            await self.hooks.fire(AgentEvent(
                type=AgentEventType.TURN_END,
                data={"turn": turn_count, "tool_calls": tool_call_count}
            ))

            # ★ Phase 3：update_working_checkpoint（每轮）
            if self.autonomous_memory:
                self.autonomous_memory.update_working_checkpoint(
                    key_info=f"Turn {turn_count}: {tool_call_count} tool calls",
                )

                # ★ Phase 3：Nudge 检查（每10轮）
                nudge = self.autonomous_memory.get_nudge_content(
                    turn_count, "memory"
                )
                if nudge:
                    context.messages.append(Message(
                        role="user", content=f"[系统提醒] {nudge}"
                    ))

            # LLM 返回了文本但没调用工具 → 可能需要 followUp
            if response_text and not tool_calls:
                # LLM 主动要求更多信息 → 等待用户输入（CLI）
                # 这里不自动 followUp，让 CLI 层处理
                break

        # 循环结束
        await self.hooks.fire(AgentEvent(type=AgentEventType.LOOP_END))

        # ★ Phase 3：start_long_term_update（会话结束时调用）
        # 由 app/cli.py 在会话结束时调用

        return AgentResult(
            status="completed" if turn_count < self.config.max_turns else "max_turns",
            messages=context.messages,
            total_turns=turn_count,
            total_tool_calls=total_tool_calls,
            total_input_tokens=total_input_tokens,
            total_output_tokens=total_output_tokens,
        )
```

**设计决策**：
- 双层循环（参考 Pi Agent 的 followUp + tool_calls 结构）—— 比单层更清晰，避免状态机
- followUp 用 `asyncio.Queue`，steer 用独立 Queue（参考需求2 的 steer 设计）—— steer 优先级高于 followUp
- 每 10 轮 Nudge 一次（固定计数器，不用小模型判断）
- Phase 2/3 依赖全部可选（`| None`）—— Phase 1 可以独立跑通
- 工具执行在 `ToolRegistry.execute_many()` 中统一处理并发/串行/预算

**常见陷阱**：
- tool_call args 是流式返回的，需要累积拼接 JSON——不完整时等待下一个 chunk
- LLM 返回 tool_call + text 混合时——需要正确解析 content block 顺序（参考 Anthropic content block 类型）
- context.messages 无限膨胀→触发 token 超限→需要在 turn_end 检查并摘要压缩（V2）

**依赖**：`ai/types.py`, `ai/provider.py`, `agent/step_outcome.py`, `agent/tool_registry.py`, `agent/hook.py`
Phase 2 额外依赖：`agent/prompt.py`
Phase 3 额外依赖：`agent/runtime.py`, `agent/autonomous_memory.py`

```python
# app/tools/agent_message.py
from ai.types import ITool, ToolDefinition, ToolParameter, ToolResult, Context


class AgentMessageTool:
    """子Agent 间通信工具。

    设计意图：
    - 主Agent → 子Agent：spawn 后通过 send_message 传递额外指令
    - 子Agent → 主Agent：回传中间发现或请求更多上下文
    - 消息通过 AgentRuntime._inboxes 异步传递，不阻塞
    """
    definition = ToolDefinition(
        name="agent_message",
        description=(
            "向另一个 Agent 发送消息。"
            "用于子Agent之间协调、向父Agent汇报中间发现、"
            "或请求父Agent提供更多上下文。"
        ),
        parameters=[
            ToolParameter(
                name="to_agent", type="string",
                description="目标 Agent ID（'main'=主Agent, 或 task_id）",
                required=True
            ),
            ToolParameter(
                name="content", type="string",
                description="消息内容（如发现、请求、摘要）",
                required=True
            ),
            ToolParameter(
                name="message_type", type="string",
                description="消息类型：'info' / 'request' / 'result'",
                required=False, default="info",
                enum=["info", "request", "result"]
            ),
        ]
    )
    is_concurrency_safe = True  # 只写 Queue，无副作用

    async def execute(self, args: dict, context: Context) -> ToolResult:
        to_agent = args["to_agent"]
        content = args["content"]
        msg_type = args.get("message_type", "info")

        # AgentRuntime 由 context.metadata 注入
        runtime = context.metadata.get("_runtime")
        if not runtime:
            return ToolResult(
                tool_name="agent_message", success=False,
                error="AgentRuntime not available (tool only works in agent context)"
            )

        current_agent = context.metadata.get("agent_id", "main")
        try:
            runtime.send_message(
                from_agent=current_agent,
                to_agent=to_agent,
                content=f"[{msg_type}] {content}",
            )
            return ToolResult(
                tool_name="agent_message", success=True,
                content=f"消息已发送到 {to_agent}"
            )
        except ValueError as e:
            return ToolResult(
                tool_name="agent_message", success=False,
                error=str(e)
            )
```

**设计决策**：
- AgentMessageTool 不在子Agent 黑名单中（允许子Agent 间通信）
- 消息通过 asyncio.Queue 投递，不持久化（会话内通信，会话结束自动清理）
- `message_type` 区分 info/request/result——方便接收方做不同处理

**DelegateTaskTool 实现**（★ 需求4 任务委托桥梁）：

```python
# app/tools/delegate_task.py
from ai.types import ITool, ToolDefinition, ToolParameter, ToolResult, Context


class DelegateTaskTool:
    """任务委托工具 —— 连接 AgentLoop 和 IAgentRuntime 的桥梁。

    设计意图：
    - LLM 判断任务适合委托时调用此工具
    - AgentLoop 检测到 delegate_task tool call → 路由到 runtime.spawn()
    - 子Agent 黑名单含此工具，防止递归委托
    - agent_type 可选：留空时 runtime.spawn() 自动调用 select_agent() 匹配
    """
    definition = ToolDefinition(
        name="delegate_task",
        description=(
            "将任务委托给一个子Agent执行。"
            "子Agent有独立的上下文和受限的工具集，可以并行工作。"
            "适用于：跨文件重构、独立子任务、需要隔离上下文的操作。"
            "子Agent 默认禁止再委托（黑名单含 delegate_task）。"
        ),
        parameters=[
            ToolParameter(
                name="task", type="string",
                description="要委托的任务描述（必须具体、完整）",
                required=True
            ),
            ToolParameter(
                name="agent_type", type="string",
                description=(
                    "代理类型：'coder' / 'reviewer' / 'explorer'。"
                    "留空则根据 task 内容自动选择最匹配的类型（Overlap Coefficient ≥ 0.3）"
                ),
                required=False
            ),
        ]
    )
    is_concurrency_safe = False  # spawn 涉及状态变更和信号量，必须串行

    async def execute(self, args: dict, context: Context) -> ToolResult:
        task = args["task"]
        agent_type = args.get("agent_type")

        # AgentRuntime 由 context.metadata 注入（AgentLoop 初始化时注入）
        runtime = context.metadata.get("_runtime")
        if not runtime:
            return ToolResult(
                tool_name="delegate_task", success=False,
                error="AgentRuntime not available (tool only works in agent context)"
            )

        try:
            result = await runtime.spawn(
                agent_type=agent_type,  # None → auto-select
                task=task,
                context={"metadata": context.metadata},
            )
            return ToolResult(
                tool_name="delegate_task", success=True,
                content=(
                    f"子Agent [{result.agent_type}] 完成 (status={result.status}):\n"
                    f"{result.output[:3000]}"  # 预览前 3000 字符，完整结果在 result.output
                )
            )
        except Exception as e:
            return ToolResult(
                tool_name="delegate_task", success=False,
                error=f"委托失败: {type(e).__name__}: {str(e)[:500]}"
            )
```

**设计决策**：
- `is_concurrency_safe = False`：spawn 涉及信号量获取和状态变更，`execute_many()` 对不安全工具串行执行，确保并发控制正确
- `agent_type` 可选：留空时由 `runtime.spawn()` 内部调用 `select_agent()` 自动匹配
- 子Agent 黑名单默认含 `delegate_task`，防止递归委托
- 结果预览截断到 3000 字符（完整结果存入 `SubAgentResult.output`）

```python
from ai.types import ITool, ToolDefinition, ToolParameter, ToolResult, Context
from pathlib import Path

class FileReadTool:
    definition = ToolDefinition(
        name="read",
        description="读取文件内容。支持分段读取（offset + limit）。",
        parameters=[
            ToolParameter(name="file_path", type="string",
                          description="绝对路径", required=True),
            ToolParameter(name="offset", type="integer",
                          description="起始行号（1-based）", required=False),
            ToolParameter(name="limit", type="integer",
                          description="读取行数", required=False),
        ]
    )
    is_concurrency_safe = True  # 只读，安全并行

    async def execute(self, args: dict, context: Context) -> ToolResult:
        path = Path(args["file_path"])
        offset = args.get("offset", 1) - 1
        limit = args.get("limit", 2000)

        try:
            if not path.exists():
                return ToolResult(tool_name="read", success=False,
                                  error=f"文件不存在: {path}")
            content = path.read_text(encoding="utf-8")
            lines = content.split("\n")
            selected = lines[offset:offset + limit]
            return ToolResult(
                tool_name="read", success=True,
                content="\n".join(selected),
            )
        except Exception as e:
            return ToolResult(tool_name="read", success=False, error=str(e))
```

- 所有路径参数必须是绝对路径——拒绝相对路径（安全考虑）

**接口定义**：

```python
# agent/skill.py
from abc import ABC, abstractmethod
from dataclasses import dataclass, field
from pathlib import Path
import yaml
import logging

logger = logging.getLogger(__name__)


# ── ABC 定义 ──

class ISkill(ABC):
    """技能抽象接口。

    技能 = 可复用的任务模式（YAML 描述 + 工具调用序列）。
    来源：手动注册（YAML）+ 自动结晶（IAutonomousMemory.crystallize()）。
    """

    @abstractmethod
    def match(self, task_description: str) -> bool:
        """判断该技能是否匹配给定任务描述。"""
        ...

    @abstractmethod
    def get_steps(self) -> list[dict]:
        """获取技能执行步骤列表 [{tool, args, description}, ...]。"""
        ...

    @property
    @abstractmethod
    def name(self) -> str: ...

    @property
    @abstractmethod
    def description(self) -> str: ...


class ISkillConfig(ABC):
    """技能配置抽象接口。

    元数据层：分类、优先级、标签、自动加载策略。
    与 ISkill 分离——同一个技能可有不同配置（如不同场景不同优先级）。
    """

    @abstractmethod
    def to_dict(self) -> dict: ...

    @abstractmethod
    def to_yaml(self, path: str) -> None: ...


# ── 数据类 ──

@dataclass
class SkillConfig(ISkillConfig):
    """ISkillConfig 实现。"""
    category: str = "general"           # coding / deploy / analysis / ...
    priority: int = 50                  # 0-100，越高越优先匹配
    tags: list[str] = field(default_factory=list)
    auto_load: bool = True              # 是否自动加载到 ISkillRegistry
    requires_confirmation: bool = False # 执行前是否需要用户确认

    def to_dict(self) -> dict:
        return {
            "category": self.category,
            "priority": self.priority,
            "tags": self.tags,
            "auto_load": self.auto_load,
            "requires_confirmation": self.requires_confirmation,
        }

    def to_yaml(self, path: str) -> None:
        Path(path).parent.mkdir(parents=True, exist_ok=True)
        with open(path, "w", encoding="utf-8") as f:
            yaml.dump(self.to_dict(), f, allow_unicode=True)


@dataclass
class Skill(ISkill):
    """ISkill 实现——单条技能卡。

    参考 GenericAgent Skill Card 格式：
    - name: 唯一标识
    - description: 自然语言描述
    - trigger_condition: 触发条件（用于 match()）
    - steps: 工具调用序列
    - quality_score: 质量评分（手动技能默认 100，自动结晶由 ISelfModification 评分）
    """
    name: str
    description: str = ""
    trigger_condition: str = ""
    steps: list[dict] = field(default_factory=list)
    config: SkillConfig = field(default_factory=SkillConfig)
    quality_score: float = 100.0       # 手动注册默认满分
    created_at: float = 0.0
    last_used_at: float = 0.0
    use_count: int = 0

    def match(self, task_description: str) -> bool:
        """基于 trigger_condition 做简单关键词匹配。
        Phase 3 可选：接入 _tokenize() 做中文分词匹配。
        """
        if not self.trigger_condition:
            return False
        keywords = self.trigger_condition.lower().split()
        task_lower = task_description.lower()
        return any(kw in task_lower for kw in keywords)

    def get_steps(self) -> list[dict]:
        return self.steps


# ── 注册表 ──

class SkillRegistry:
    """ISkill 注册表。

    管理技能的生命周期：注册 → 查找 → 匹配 → 卸载。
    Phase 1: 手动注册 + YAML 目录加载
    Phase 3: IAutonomousMemory 结晶自动注册
    """

    def __init__(self):
        self._skills: dict[str, Skill] = {}

    def register(self, skill: Skill) -> None:
        """注册技能。名称重复时覆盖（更新）。"""
        if skill.name in self._skills:
            logger.info(f"Skill '{skill.name}' already registered, overwriting")
        self._skills[skill.name] = skill

    def unregister(self, name: str) -> None:
        self._skills.pop(name, None)

    def get(self, name: str) -> Skill | None:
        return self._skills.get(name)

    def find_by_tags(self, tags: list[str]) -> list[Skill]:
        return [s for s in self._skills.values()
                if any(t in s.config.tags for t in tags)]

    def match_task(self, task: str) -> list[Skill]:
        """返回匹配技能列表，按 priority 降序。"""
        matched = [s for s in self._skills.values() if s.match(task)]
        matched.sort(key=lambda s: s.config.priority, reverse=True)
        return matched

    def list_all(self) -> list[Skill]:
        return list(self._skills.values())

    def load_from_dir(self, dir_path: str) -> int:
        """从目录加载 YAML 技能卡。

        技能卡格式（.yaml）：
          name / description / trigger_condition / steps / config / quality_score
        参考 Hermes 技能 YAML manifest。
        """
        import os
        count = 0
        skill_dir = Path(dir_path)
        if not skill_dir.exists():
            return 0
        for yaml_file in skill_dir.glob("*.yaml"):
            try:
                with open(yaml_file, "r", encoding="utf-8") as f:
                    data = yaml.safe_load(f)
                config_data = data.pop("config", {})
                skill = Skill(
                    name=data.get("name", yaml_file.stem),
                    description=data.get("description", ""),
                    trigger_condition=data.get("trigger_condition", ""),
                    steps=data.get("steps", []),
                    config=SkillConfig(**config_data) if config_data else SkillConfig(),
                    quality_score=data.get("quality_score", 100.0),
                    created_at=data.get("created_at", 0.0),
                )
                self.register(skill)
                count += 1
            except Exception as e:
                logger.warning(f"Failed to load skill from {yaml_file}: {e}")
        return count
```

**设计决策**：
- ISkill 和 ISkillConfig 分离——前者定义"做什么"，后者定义"何时/如何用"

```python
# agent/prompt.py
from dataclasses import dataclass, field
from typing import Protocol, Callable
from ai.types import Context


@dataclass
class PromptFragment:
    """一个 prompt 片段。"""
    name: str                          # 唯一标识，如 "CAPABILITY_OVERVIEW"
    content: str                       # 片段文本
    priority: int = 50                 # 0-100，越小越核心（0-24 不可裁剪）
    condition: Callable[[Context], bool] | None = None  # 条件注入函数
    source: str = ""                   # "builtin" / "plugin:{name}"


class IPromptBuilder(Protocol):
    """动态系统提示词构建器。

    参考 oh-my-opencode 的片段注册 + token 预算模式。
    """

    def register_fragment(self, fragment: PromptFragment) -> None: ...
    def unregister_fragment(self, name: str) -> None: ...
    def build(self, context: Context, max_tokens: int = 8000) -> str:
        """按优先级组装片段，超 token 预算则裁剪低优先级片段。"""
        ...

    def refresh_fragments(self, trigger: str) -> None:
        """事件驱动刷新：plugin_loaded / memory_updated / turn_end 时调用。"""
        ...

    def get_token_usage(self) -> dict:
        """返回各片段的 token 用量统计。"""
        ...


class PromptBuilder:
    """IPromptBuilder 的实现。

    组装逻辑：
    1. 按 priority 升序排列片段
    2. 核心片段（0-24）无条件保留
    3. 条件片段检查 condition(context)，不满足则跳过
    4. 从低优先级片段开始裁剪，直到总 token 数 <= max_tokens
    """

    def __init__(self):
        self._fragments: dict[str, PromptFragment] = {}
        self._cached_result: str | None = None
        self._cached_hash: str = ""

    def build(self, context: Context, max_tokens: int = 8000) -> str:
        fragments = sorted(
            [f for f in self._fragments.values()
             if f.condition is None or f.condition(context)],
            key=lambda f: f.priority
        )

        # 组装
        parts = []
        token_count = 0

        for f in fragments:
            frag_tokens = len(f.content) // 3  # 粗略计数
            if f.priority <= 24:
                # 核心片段，必须保留
                parts.append(f.content)
                token_count += frag_tokens
            elif token_count + frag_tokens <= max_tokens:
                parts.append(f.content)
                token_count += frag_tokens
            # else: 裁剪

        return "\n\n".join(parts)
```

**实现内容**：

```python
# agent/claudemd.py
from pathlib import Path
from dataclasses import dataclass, field


@dataclass
class ClaudeMdFile:
    """一个 CLAUDE.md 文件及其元数据。"""
    path: str                          # 文件绝对路径
    content: str                       # 文件内容
    level: str                         # "user" | "project"
    priority: int                      # 数字越小优先级越低（根目录=0，当前目录=最高）


def discover_claude_mds(
    cwd: str | None = None,
    user_home: str | None = None,
) -> list[ClaudeMdFile]:
    """从项目层级中发现所有 CLAUDE.md 文件。

    发现顺序（优先级从低到高）：
    1. 用户级：~/.agent/CLAUDE.md（全局行为指南）
    2. 项目级：从项目根目录向 CWD 逐级向上遍历
       - 每级目录检查：{dir}/CLAUDE.md 和 {dir}/.agent/CLAUDE.md

    遍历逻辑（参考 CCB claudemd.ts getMemoryFiles）：
    - 从 CWD 向上逐级获取父目录，直到文件系统根目录
    - 反转目录列表（根目录在前，CWD 在最后）
    - 对每个目录检查 CLAUDE.md 是否存在
    - 后发现的文件优先级更高（覆盖前面的）

    V1 简化：
    - 不做 @include 指令解析
    - 不做 frontmatter 解析
    - 不做文件类型过滤
    - 不做条件规则（paths glob）
    - 文件大小限制：单文件最大 50KB（防止过大文件撑爆上下文）
    """
    cwd = Path(cwd or Path.cwd()).resolve()
    user_home = Path(user_home or Path.home())

    results: list[ClaudeMdFile] = []

    # 1. 用户级 CLAUDE.md（最低优先级，优先加载）
    user_claude_md = user_home / ".agent" / "CLAUDE.md"
    if user_claude_md.exists():
        content = _read_safe(user_claude_md)
        if content:
            results.append(ClaudeMdFile(
                path=str(user_claude_md),
                content=content,
                level="user",
                priority=0,
            ))

    # 2. 项目级：从 CWD 向上遍历
    project_root = _find_project_root(cwd)
    directories = _collect_directories(cwd, project_root)

    for idx, directory in enumerate(directories):
        # 检查 {dir}/CLAUDE.md
        claude_md = directory / "CLAUDE.md"
        if claude_md.exists():
            content = _read_safe(claude_md)
            if content:
                results.append(ClaudeMdFile(
                    path=str(claude_md),
                    content=content,
                    level="project",
                    priority=idx + 1,     # 根目录=1，CWD=最高
                ))

        # 检查 {dir}/.agent/CLAUDE.md
        agent_claude_md = directory / ".agent" / "CLAUDE.md"
        if agent_claude_md.exists():
            content = _read_safe(agent_claude_md)
            if content:
                results.append(ClaudeMdFile(
                    path=str(agent_claude_md),
                    content=content,
                    level="project",
                    priority=idx + 1,
                ))

    # 按 priority 升序排列（低级优先，高级覆盖）
    results.sort(key=lambda f: f.priority)
    return results


def assemble_claude_md_content(files: list[ClaudeMdFile]) -> str:
    """将多个 CLAUDE.md 文件组装为单个上下文字符串。

    组装格式（参考 CCB getClaudeMds 的输出格式）：
    ```
    # CLAUDE.md

    Contents of ~/.agent/CLAUDE.md (user's global instructions):

    <content>

    ---

    Contents of /project/CLAUDE.md (project instructions):

    <content>
    ```

    每个文件之间用 --- 分隔。
    """
    if not files:
        return ""

    parts = ["# CLAUDE.md\n"]
    for f in files:
        if f.level == "user":
            desc = "user's global instructions for all projects"
        else:
            desc = "project instructions, checked into the codebase"

        parts.append(f"Contents of {f.path} ({desc}):\n")
        parts.append(f.content)
        parts.append("")  # 空行分隔

    return "\n".join(parts)


def _find_project_root(cwd: Path) -> Path:
    """查找项目根目录（.git 所在目录或文件系统根目录）。"""
    current = cwd
    while current != current.parent:
        if (current / ".git").exists():
            return current
        current = current.parent
    return cwd  # 找不到 .git 时，以 CWD 为根


def _collect_directories(cwd: Path, project_root: Path) -> list[Path]:
    """从 project_root 到 CWD 的目录列表（根目录在前，CWD 在最后）。"""
    directories = []
    current = cwd
    while current != project_root.parent:
        directories.append(current)
        if current == project_root:
            break
        current = current.parent
    directories.reverse()  # 根目录在前，CWD 在最后
    return directories


def _read_safe(path: Path, max_bytes: int = 50_000) -> str | None:
    """安全读取文件。超过大小限制返回 None，编码错误返回 None。"""
    try:
        if path.stat().st_size > max_bytes:
            return None  # 文件过大，跳过
        return path.read_text(encoding="utf-8")
    except Exception:
        return None
```

```python
# agent/inspector.py
import asyncio
from collections import deque
from dataclasses import dataclass, field
from typing import Protocol


@dataclass
class InspectionSnapshot:
    """一次检查快照。"""
    turn_count: int
    tool_success_rate: float          # 工具成功率
    avg_llm_latency_ms: float         # 平均 LLM 延迟
    total_input_tokens: int
    total_output_tokens: int
    active_tools: list[str]
    timestamp: float


class IFlowInspector(Protocol):
    """运行流程检测器。"""

    async def push(self, event: dict) -> None:
        """推送事件（put_nowait，队列满静默丢弃）。"""
        ...

    async def start(self, interval_sec: int = 5) -> None:
        """启动旁路协程：drain → batch → analyze → write。"""
        ...

    async def get_recent_stats(self, window: int = 100) -> InspectionSnapshot:
        """获取滑动窗口统计。"""
        ...

    async def stop(self) -> None: ...


class FlowInspector:
    """IFlowInspector 实现。

    旁路模式：
    1. Agent 循环通过 push() 推送事件（非阻塞）
    2. 独立协程定期 drain 队列 → 聚合计算 → 写入文件
    3. 前端（CLI/Web UI）通过 get_recent_stats() 轮询

    滑动窗口：最近 100 轮
    """

    def __init__(self, queue_size: int = 1000):
        self._queue: asyncio.Queue[dict] = asyncio.Queue(maxsize=queue_size)
        self._window: deque[dict] = deque(maxlen=100)
        self._running = False

    async def push(self, event: dict) -> None:
        """非阻塞入队。队列满时静默丢弃。"""
        try:
            self._queue.put_nowait(event)
        except asyncio.QueueFull:
            pass  # 静默丢弃

    async def start(self, interval_sec: int = 5) -> None:
        """启动旁路消费协程。"""
        self._running = True
        while self._running:
            # drain 队列
            batch = []
            while not self._queue.empty():
                try:
                    batch.append(self._queue.get_nowait())
                except asyncio.QueueEmpty:
                    break
            # 聚合到滑动窗口
            self._window.extend(batch)
            # 等待下一轮
            await asyncio.sleep(interval_sec)

    async def get_recent_stats(self, window: int = 100) -> InspectionSnapshot:
        recent = list(self._window)[-window:]
        if not recent:
            return InspectionSnapshot(
                turn_count=0, tool_success_rate=1.0,
                avg_llm_latency_ms=0, total_input_tokens=0,
                total_output_tokens=0, active_tools=[], timestamp=0
            )
        successes = sum(1 for e in recent if e.get("success"))
        return InspectionSnapshot(
            turn_count=len(recent),
            tool_success_rate=successes / len(recent),
            avg_llm_latency_ms=sum(
                e.get("latency_ms", 0) for e in recent
            ) / len(recent),
            total_input_tokens=sum(e.get("input_tokens", 0) for e in recent),
            total_output_tokens=sum(e.get("output_tokens", 0) for e in recent),
            active_tools=list(set(
                e.get("tool_name", "") for e in recent if e.get("tool_name")
            )),
            timestamp=recent[-1].get("timestamp", 0),
        )

    async def stop(self) -> None:
        self._running = False
```

```python
class AgentRuntime(IAgentRuntime):
    """IAgentRuntime 实现。

    V1 采用 Hermes 线程级隔离（asyncio），参考 multica 信号量并发控制。

    并发安全规则（参考 Hermes delegate_tool.py skip_memory=True）：
    1. Nudge 只在主Agent的 turn_end 触发
    2. 子Agent的 tools_blacklist 默认含 memory 工具
    3. 子Agent不触发、不接收、不响应 Nudge
    """

    def __init__(
        self,
        tools: dict,                    # 父Agent的完整工具集 {name: ITool}
        provider: "IModelProvider",     # 父Agent的模型提供商（子Agent复用）
        hooks: HookChain,
        max_concurrent: int = 3,        # 参考 Hermes max_concurrent_children=3
        max_depth: int = 2,             # 参考 Hermes MAX_DEPTH=2
        default_timeout: int = 300,
    ):
        self._tools = tools
        self._provider = provider
        self._hooks = hooks
        self._max_concurrent = max_concurrent
        self._semaphore = asyncio.Semaphore(max_concurrent)
        self._max_depth = max_depth
        self._default_timeout = default_timeout
        self._agent_types: dict[str, AgentTypeConfig] = {}
        self._active_children: dict[str, dict] = {}
        self._inboxes: dict[str, asyncio.Queue] = {}   # ★ Agent间通信收件箱
        self._stats = {
            "total_spawned": 0, "total_completed": 0, "total_failed": 0,
            "total_input_tokens": 0, "total_output_tokens": 0,
            "total_cost": 0.0, "max_depth_reached": 0,
        }

    # ── 代理类型注册 ──

    def register_agent_type(self, config: AgentTypeConfig) -> None:
        """注册代理类型。持久化到 .agent-base/agents/{name}.yaml。

        参考 CCB loadAgentsDir.ts 的 6 层覆盖机制。
        V1 简化：直接注册 + 目录扫描。
        """
        if config.name in self._agent_types:
            raise ValueError(f"Agent type '{config.name}' already registered")
        # 验证 tools_whitelist 引用的工具存在
        for tn in config.tools_whitelist:
            if tn not in self._tools:
                raise ValueError(f"Tool '{tn}' in whitelist not found in catalog")
        self._agent_types[config.name] = config
        # 持久化
        config.to_yaml(f".agent-base/agents/{config.name}.yaml")

    def get_agent_type(self, name: str) -> AgentTypeConfig | None:
        return self._agent_types.get(name)

    def list_agent_types(self) -> list[AgentTypeConfig]:
        return list(self._agent_types.values())

    # ── 工具过滤 ──

    def _filter_tools(self, config: AgentTypeConfig) -> dict:
        """黑名单优先 → 白名单筛选 → 返回过滤后的工具集。

        参考 Hermes DELEGATE_BLOCKED_TOOLS + 工具集过滤。
        路径：controable code\hermes-agent-main\hermes-agent-main\tools\delegate_tool.py:33-50
        """
        filtered = {}
        for name, tool in self._tools.items():
            if name in config.tools_blacklist:
                continue
            if config.tools_whitelist and name not in config.tools_whitelist:
                continue
            filtered[name] = tool
        return filtered

    # ── Agent间通信 ──

    def send_message(self, from_agent: str, to_agent: str, content: str) -> None:
        """向指定 Agent 发送消息。

        消息通过 asyncio.Queue 投递，非阻塞。
        每个 Agent 在 spawn 时自动创建 inbox Queue。
        """
        if to_agent not in self._inboxes:
            self._inboxes[to_agent] = asyncio.Queue()
        try:
            self._inboxes[to_agent].put_nowait({
                "from": from_agent, "content": content,
                "timestamp": time.time()
            })
        except asyncio.QueueFull:
            pass  # 收件箱满，静默丢弃

    def check_inbox(self, agent_id: str) -> str | None:
        """检查指定 Agent 的收件箱，返回最早未读消息。非阻塞。"""
        inbox = self._inboxes.get(agent_id)
        if not inbox or inbox.empty():
            return None
        try:
            msg = inbox.get_nowait()
            return msg["content"]
        except asyncio.QueueEmpty:
            return None

    # ── 工具函数 ──

    @staticmethod
    def _tokenize(text: str) -> list[str]:
        """中文分词 + 英文分词统一入口。

        优先使用 jieba 分词（中文友好），jieba 不可用时回退到字符二元组。
        规范副本位于 agent/memory/index.py 的 MemoryIndex._tokenize()。
        此方法为供 select_agent() 使用的副本，两者须保持一致。
        """
        try:
            import jieba
            import re
            words = jieba.lcut(text.lower())
            return [w.strip() for w in words
                    if w.strip() and not re.match(r'^[\s\W_]+$', w)]
        except ImportError:
            pass
        import re
        clean = text.lower().strip()
        tokens = []
        eng_words = re.findall(r'[a-z0-9]+', clean)
        tokens.extend(eng_words)
        chinese_chars = re.findall(r'[一-鿿]+', clean)
        for segment in chinese_chars:
            for i in range(len(segment) - 1):
                tokens.append(segment[i:i+2])
            if len(segment) == 1:
                tokens.append(segment)
        return tokens

    # ── Agent 自动选择 ──

    def select_agent(self, task: str) -> str | None:
        """根据任务描述自动选择最匹配的 Agent 类型。

        匹配算法：Overlap Coefficient —— |A ∩ B| / min(|A|, |B|)
        对中文友好：通过 _tokenize() 做中文分词后再计算。

        流程：
        1. 对 task 分词 → task_tokens
        2. 遍历所有注册的 AgentTypeConfig：
           a. 对 name + description + when_to_use + category 分词 → agent_tokens
           b. 计算 overlap = |task_tokens ∩ agent_tokens| / min(|task_tokens|, |agent_tokens|)
        3. 返回 overlap 最高且 >= 0.3 的 agent name
        4. 如果所有 < 0.3，返回 None（由 LLM 自行决定或使用默认类型）
        """
        if not self._agent_types or not task:
            return None

        task_tokens = set(self._tokenize(task))
        if not task_tokens:
            return None

        best_name = None
        best_score = 0.0

        for name, config in self._agent_types.items():
            # 构建匹配文本
            match_text = " ".join([
                name,
                config.description,
                config.when_to_use,
                config.category,
            ])
            agent_tokens = set(self._tokenize(match_text))
            if not agent_tokens:
                continue

            intersection = task_tokens & agent_tokens
            overlap = len(intersection) / min(len(task_tokens), len(agent_tokens))

            if overlap > best_score:
                best_score = overlap
                best_name = name

        return best_name if best_score >= 0.3 else None

    # ── spawn ──

    async def spawn(
        self, agent_type: str | None = None, task: str = "",
        context: dict | None = None,
        current_depth: int = 0
    ) -> SubAgentResult:
        """派生子Agent执行任务。

        参考 Hermes delegate_tool.py 709-716行 的 spawn 流程。
        参考 CCB forkSubagent.ts 78-89行 的递归检测。
        参考 multica daemon.go 的信号量并发控制。

        agent_type 可选：None 时自动调用 select_agent(task) 选择最匹配类型。
        """
        task_id = uuid4().hex[:8]

        # 0. Agent 自动选择（agent_type 为空时）
        if not agent_type:
            agent_type = self.select_agent(task)
            if not agent_type:
                return SubAgentResult(
                    task_id=task_id, agent_type="unknown",
                    status="rejected", output="",
                    usage={}, duration_ms=0,
                    error="agent_type not specified and auto-selection found no match"
                )

        # 0. 深度检查（参考 Hermes MAX_DEPTH=2）
        if current_depth >= self._max_depth:
            return SubAgentResult(
                task_id=task_id, agent_type=agent_type,
                status="rejected", output="",
                usage={}, duration_ms=0,
                error=f"Max depth reached ({self._max_depth})"
            )

        # 1. 获取配置
        config = self.get_agent_type(agent_type)
        if not config:
            return SubAgentResult(
                task_id=task_id, agent_type=agent_type,
                status="failed", output="",
                usage={}, duration_ms=0,
                error=f"Unknown agent type: {agent_type}"
            )

        # 1.5 为当前 agent_id 创建 inbox（如果尚未存在）
        agent_id = context.get("agent_id", task_id) if context else task_id
        if agent_id not in self._inboxes:
            self._inboxes[agent_id] = asyncio.Queue(maxsize=50)

        # 2. 并发控制（参考 multica 信号量）
        async with self._semaphore:
            # 3. 过滤工具
            filtered_tools = self._filter_tools(config)

            # 4. 触发 subagent_start hook
            await self._hooks.fire(AgentEvent(
                type=AgentEventType.SUBAGENT_START,
                data={"task_id": task_id, "agent_type": agent_type, "task": task}
            ))

            # 5. 构建隔离上下文（task 不预添加，由 AgentLoop.run() 统一处理）
            child_context = Context(
                system_prompt=config.system_prompt,
                tools=filtered_tools,
                metadata={
                    "task_id": task_id,
                    "agent_type": agent_type,
                    "agent_id": task_id,       # ★ 子Agent 标识（收件箱用）
                    "_runtime": self,          # ★ 注入 runtime 引用（AgentMessageTool 用）
                    "depth": current_depth + 1,
                    "parent_messages": context.get("parent_messages", []) if context else [],
                }
            )

            # 注入 initial_prompt（参考 CCB buildChildMessage）
            if config.initial_prompt:
                child_context.messages.append(Message(
                    role="user", content=config.initial_prompt
                ))

            # 6. 执行子Agent循环（复用 agent/loop.py）
            start_time = time.time()
            self._active_children[task_id] = {
                "task_id": task_id, "agent_type": agent_type,
                "status": "running", "started_at": start_time
            }
            self._stats["total_spawned"] += 1

            try:
                # 注意：懒加载导入 AgentLoop，避免循环引用
                from agent.loop import AgentLoop, AgentConfig
                # 将过滤后的 dict 转为 ToolRegistry（AgentLoop 的入参类型）
                child_registry = ToolRegistry()
                for name, tool in filtered_tools.items():
                    child_registry.register(tool)
                child_loop = AgentLoop(
                    provider=self._provider,
                    tools=child_registry,
                    hooks=self._hooks,
                    config=AgentConfig(max_turns=50),  # 子Agent限制更严
                )
                result = await asyncio.wait_for(
                    child_loop.run(task, child_context),
                    timeout=self._default_timeout
                )
                status = "completed"
                error = None
                self._stats["total_completed"] += 1

            except asyncio.TimeoutError:
                status = "timeout"
                error = f"Timeout after {self._default_timeout}s"
                result = None
                self._stats["total_failed"] += 1

            except Exception as e:
                status = "failed"
                error = str(e)
                result = None
                self._stats["total_failed"] += 1

            finally:
                await self._cleanup(task_id)

            duration_ms = int((time.time() - start_time) * 1000)

            # 7. 触发 subagent_stop hook
            await self._hooks.fire(AgentEvent(
                type=AgentEventType.SUBAGENT_STOP,
                data={
                    "task_id": task_id,
                    "agent_type": agent_type,
                    "status": status,
                    "usage": result.total_input_tokens if result else {},
                    "duration_ms": duration_ms,
                }
            ))

            return SubAgentResult(
                task_id=task_id,
                agent_type=agent_type,
                status=status,
                output=result.messages[-1].content if result and result.messages else "",
                usage={
                    "input_tokens": result.total_input_tokens if result else 0,
                    "output_tokens": result.total_output_tokens if result else 0,
                },
                duration_ms=duration_ms,
                error=error,
                exit_reason=result.status if result else "error",
            )

    # ── spawn_parallel ──

    async def spawn_parallel(
        self, tasks: list[dict], max_concurrency: int = 3
    ) -> list[SubAgentResult]:
        """并行派生多个子Agent。

        参考 Pi Agent mapWithConcurrencyLimit(tasks, 4)。
        参考 multica 有界信号量 pollLoop。
        """
        sem = asyncio.Semaphore(max_concurrency)

        async def bounded_spawn(task_dict):
            async with sem:
                return await self.spawn(**task_dict)

        return await asyncio.gather(*[bounded_spawn(t) for t in tasks])

    # ── 状态查询 ──

    def get_active_children(self) -> list[dict]:
        return [
            v for v in self._active_children.values()
            if v["status"] == "running"
        ]

    def get_delegation_stats(self) -> dict:
        return {**self._stats, "active_count": len(self.get_active_children())}

    def get_config(self) -> dict:
        return {
            "max_concurrent_children": self._max_concurrent,
            "max_depth": self._max_depth,
            "default_timeout_sec": self._default_timeout,
        }

    # ── 清理 ──

    async def _cleanup(self, task_id: str) -> None:
        """资源清理（参考 Hermes delegate_tool.py 643-684行 finally 块）。

        清理顺序：
        1. 取消子Agent未完成异步任务
        2. 关闭子Agent工具连接
        3. 清理记忆命名空间
        4. 从 active_children 移除
        """
        self._active_children.pop(task_id, None)
```

**参考**：

**实现内容**：

```python
# agent/autonomous_memory.py
import time
import yaml
from pathlib import Path
from dataclasses import dataclass, field

from agent.memory.store import MemoryStore


@dataclass
class SkillConfig:
    """技能配置（ISkillConfig 实现）。

    持久化位置：.agent-memory/{project}/skills/{name}.yaml
    """
    name: str
    description: str
    trigger_condition: str
    steps: list[dict]                  # [{tool, args, description}, ...]
    quality_score: float = 0.0         # ISelfModification.quality_score() 评分
    created_at: float = 0.0
    last_used_at: float = 0.0
    use_count: int = 0


class IAutonomousMemory(ABC):
    """自主记忆抽象接口。

    管理 Agent 的会话内工作检查点、跨会话长期更新、技能结晶和 Nudge 提醒。
    参考 GenericAgent L0-L4 记忆公理 + Hermes Nudge 机制。
    """

    @abstractmethod
    def update_working_checkpoint(self, key_info: str, project: str = "default", related_sop: str | None = None) -> None:
        """更新工作检查点。每轮 turn_end 调用。"""
        ...

    @abstractmethod
    def get_working_checkpoint(self, project: str = "default") -> dict:
        """获取工作检查点内容。"""
        ...

    @abstractmethod
    def should_crystallize(self, task_result: dict) -> bool:
        """评估任务是否值得结晶为技能。"""
        ...

    @abstractmethod
    def crystallize(self, task_result: dict) -> "SkillCard | None":
        """将成功任务结晶为技能卡。"""
        ...

    @abstractmethod
    def get_nudge_content(self, turn_count: int, nudge_type: str = "memory") -> str | None:
        """获取 Nudge 提醒内容。每 10 轮触发一次。"""
        ...

    @abstractmethod
    def record_delegation_result(self, task_id: str, result: dict) -> None:
        """记录子Agent 委托结果，用于长期沉淀。"""
        ...

    @abstractmethod
    def start_long_term_update(self, project: str = "default") -> None:
        """启动跨会话长期更新（会话结束时调用）。"""
        ...


class AutonomousMemory(IAutonomousMemory):
    """IAutonomousMemory 实现。

    参考 GenericAgent 的结晶理念 + Hermes 的 Nudge 机制。

    核心公理（来自 GenericAgent）：
    - No Execution, No Memory
    - 神圣不可删改
    - 禁止存储易变状态
    - 最小充分指针
    """

    def __init__(self, store: MemoryStore, agent_id: str = "default"):
        self.store = store
        self.agent_id = agent_id
        self._checkpoint_path = f"{{project}}/agents/{agent_id}/working_checkpoint.md"
        self._skills_dir = "{project}/skills"
        self._last_crystallized: set[str] = set()  # 去重：已结晶的任务摘要

    # ── 工作检查点 ──

    def update_working_checkpoint(self, key_info: str,
                                   related_sop: str = "",
                                   project: str = "default") -> None:
        """更新工作检查点。

        参考 GenericAgent update_working_checkpoint()。
        工作记忆在每轮 prompt 中通过 WORKING_CHECKPOINT 片段注入。
        """
        content = f"""# Working Checkpoint
Last updated: {time.strftime("%Y-%m-%d %H:%M:%S")}

## Current Task
{key_info}

## Related SOP
{related_sop or "None"}
"""
        path = self._checkpoint_path.format(project=project)
        self.store.write(path, content)

    def get_working_checkpoint(self, project: str = "default") -> dict:
        """获取工作检查点。"""
        path = self._checkpoint_path.format(project=project)
        content = self.store.read(path)
        if content:
            return {"key_info": content, "last_updated": time.time()}
        return {"key_info": "", "last_updated": 0}

    # ── 结晶触发 ──

    def should_crystallize(self, task_result: dict) -> bool:
        """评估任务是否值得结晶。

        启发式规则（不调 LLM）：
        1. success == True
        2. tool_calls_count >= 3（复杂度足够）
        3. duration_ms >= 5000（非平凡任务）
        4. task_description 不在最近结晶集中（去重）
        """
        if not task_result.get("success"):
            return False
        if task_result.get("tool_calls_count", 0) < 3:
            return False
        if task_result.get("duration_ms", 0) < 5000:
            return False
        desc = task_result.get("task_description", "")
        if desc in self._last_crystallized:
            return False
        return True

    async def crystallize(self, task_result: dict) -> SkillConfig | None:
        """结晶为技能。

        流程（参考 GenericAgent）：
        1. 提取工具调用序列
        2. 生成技能描述
        3. 调用 ISelfModification.quality_score() 评估
        4. score >= 60 → 持久化到 .agent-memory/{project}/skills/{name}.yaml
        5. score < 60 → 丢弃（记日志）
        """
        tool_sequence = task_result.get("tool_sequence", [])
        task_desc = task_result.get("task_description", "Unknown task")

        # 生成技能名（简化：从任务描述取前几个词）
        skill_name = "_".join(task_desc.split()[:4]).lower().replace(" ", "_")

        skill = SkillConfig(
            name=skill_name,
            description=task_desc,
            trigger_condition=f"当需要{task_desc}时",
            steps=[
                {"tool": ts["tool_name"], "args": ts.get("args", {}),
                 "description": ts.get("description", "")}
                for ts in tool_sequence
            ],
            created_at=time.time(),
        )

        # 质量评分（从外部注入 ISelfModification）
        if hasattr(self, '_quality_scorer'):
            score_result = self._quality_scorer(skill)
            skill.quality_score = score_result.get("quality_score", 0)

        if skill.quality_score >= 60:
            # 持久化
            skill_path = f"{self._skills_dir}/{skill_name}.yaml"
            skill_data = {
                "name": skill.name,
                "description": skill.description,
                "trigger_condition": skill.trigger_condition,
                "steps": skill.steps,
                "quality_score": skill.quality_score,
                "created_at": skill.created_at,
            }
            self.store.write(
                skill_path.format(project="default"),
                yaml.dump(skill_data, allow_unicode=True)
            )
            self._last_crystallized.add(task_desc)
            return skill
        else:
            return None  # 质量不达标，丢弃

    # ── Nudge ──

    def get_nudge_content(self, turn_count: int,
                           nudge_type: str = "memory") -> str | None:
        """Nudge 提示内容。

        触发条件：
        - memory nudge：每 10 轮
        - skill nudge：每 10 轮（与 memory 错开 5 轮）
        - crystallize nudge：任务成功完成时（外部调用）

        V1：固定计数器。V3 预留：用 LLM 判断时机。
        """
        if nudge_type == "memory" and turn_count % 10 == 0:
            return "你应该考虑使用记忆工具检查或存储相关记忆。"
        if nudge_type == "skill" and turn_count % 10 == 5:
            return "你应该考虑检查是否有可复用的已结晶技能。"
        return None

    # ── 长期更新 ──

    async def start_long_term_update(self, session_summary: dict,
                                      project: str = "default") -> None:
        """会话结束后更新长期记忆。

        参考 GenericAgent start_long_term_update()。
        遵循核心公理：只有工具验证过的才记。
        """
        # 提取环境事实 → L2
        if env_facts := session_summary.get("environment_facts"):
            facts_path = f"{project}/agents/{self.agent_id}/facts.md"
            existing = self.store.read(facts_path) or ""
            self.store.write(facts_path, existing + "\n" + env_facts)

        # 提取任务经验 → L3
        if task_lessons := session_summary.get("task_lessons"):
            lessons_path = f"{project}/agents/{self.agent_id}/sessions/lessons.md"
            existing = self.store.read(lessons_path) or ""
            self.store.write(lessons_path, existing + "\n" + task_lessons)

    # ── 子Agent结果沉淀 ──

    def record_delegation_result(self, task: str,
                                   result: "SubAgentResult") -> None:
        """记录子Agent结果。

        参考 Hermes MemoryManager.on_delegation()。
        子Agent的发现（文件结构、依赖版本等）沉淀到主Agent记忆。
        """
        content = f"""## Delegation Result
Task: {task}
Agent: {result.agent_type}
Status: {result.status}
Output: {result.output[:1000]}
Usage: {result.usage}
Tool trace: {result.tool_trace}
"""
        path = f"default/agents/{self.agent_id}/delegations/{result.task_id}.md"
        self.store.write(path, content)

    # ── 技能查询 ──

    def get_crystallized_skill(self, name: str,
                                project: str = "default") -> SkillConfig | None:
        """按名称查询已结晶技能。"""
        skill_path = f"{self._skills_dir.format(project=project)}/{name}.yaml"
        content = self.store.read(skill_path)
        if content:
            data = yaml.safe_load(content)
            return SkillConfig(**data)
        return None
```

**设计决策**：
- `should_crystallize()` 用启发式规则（不调 LLM）——成功 + 3+ 工具 + 耗时 + 去重
- 质量门槛：`quality_score >= 60` 才持久化
- Nudge 用固定计数器（每 10 轮），不用小模型判断（V3 预留）
- 只有主 Agent 触发 Nudge，子 Agent 不触发
- 技能卡持久化为 YAML（结构化 + 可注释 + Python 原生加载）

```python
# agent/self_modify.py
from dataclasses import dataclass


@dataclass
class QualityScore:
    """三维质量评分结果。"""
    clarity: float             # 0-100：描述是否清晰
    completeness: float        # 0-100：步骤是否完整
    actionability: float       # 0-100：是否可直接执行
    quality_score: float       # 综合分：clarity*0.3 + completeness*0.3 + actionability*0.4
    passed: bool               # quality_score >= 60


def calculate_quality_score(skill_config: "SkillConfig") -> QualityScore:
    """基于规则的启发式评分（V1）。

    参考 GenericAgent 的三维评分体系。

    评分规则：
    - clarity：描述长度（>20字=80, >10字=50, ≤10字=20）
                + 触发条件非空（+20）
    - completeness：步骤数（≥3=80, ≥1=50, 0=10）
                     + 每步有 description（+10/步，最多+30）
    - actionability：每步有 tool_name（+25/步，最多+75）
                      + 步骤数≥2（+25）
    """
    # clarity
    desc_len = len(skill_config.description or "")
    if desc_len > 20:
        clarity = 80
    elif desc_len > 10:
        clarity = 50
    else:
        clarity = 20
    if skill_config.trigger_condition:
        clarity = min(100, clarity + 20)

    # completeness
    step_count = len(skill_config.steps or [])
    if step_count >= 3:
        completeness = 80
    elif step_count >= 1:
        completeness = 50
    else:
        completeness = 10
    described_steps = sum(
        1 for s in (skill_config.steps or [])
        if s.get("description")
    )
    completeness = min(100, completeness + described_steps * 10)

    # actionability
    actionable_steps = sum(
        1 for s in (skill_config.steps or [])
        if s.get("tool_name")
    )
    actionability = min(75, actionable_steps * 25)
    if step_count >= 2:
        actionability = min(100, actionability + 25)

    quality_score = clarity * 0.3 + completeness * 0.3 + actionability * 0.4

    return QualityScore(
        clarity=clarity,
        completeness=completeness,
        actionability=actionability,
        quality_score=round(quality_score, 1),
        passed=quality_score >= 60,
    )

class ISelfModification:
    """Agent 自我修改接口。需求2 预留 → 需求4 部分升级（新增 quality_score）。"""

    @staticmethod
    def quality_score(candidate: "SkillConfig") -> dict:
        """V1 实现：基于规则的启发式评分。"""
        result = calculate_quality_score(candidate)
        return {
            "clarity": result.clarity,
            "completeness": result.completeness,
            "actionability": result.actionability,
            "quality_score": result.quality_score,
            "pass": result.passed,
        }

    # ── 以下留 V4 ──
    async def evaluate_performance(self, session_history: dict) -> dict: ...
```

### 7.2 关键契约

**⓪ claudemd.py ↔ loop.py ↔ prompt.py**：
- loop.py 在会话初始化时调用 `discover_claude_mds()` → 结果缓存到 context.metadata
- loop.py 将 CLAUDE.md 内容注册为 PromptBuilder 的 `CLAUDE_MD` 片段（priority=0）
- claudemd.py 为纯函数模块，不持有状态，不依赖项目内任何模块（仅标准库 pathlib）

**① loop.py ↔ runtime.py**：
- loop.py 在 turn_end 时检查 Agent 是否请求委托（通过 tool call `delegate_task`）
- tool call `delegate_task` → runtime.spawn() 执行
- runtime 返回 SubAgentResult → 结果注入 context.messages
- ★ 通信：loop.py 内层循环开始时调用 `runtime.check_inbox(agent_id)` → 有新消息则注入 context
- ★ 自动选择：spawn() 的 agent_type 可选，None 时自动调用 `runtime.select_agent(task)`

**② loop.py ↔ autonomous_memory.py**：
- loop.py 每轮 turn_end 调用 `update_working_checkpoint()` + `get_nudge_content()`
- loop.py 在任务完成时调用 `should_crystallize()` → `crystallize()`
- autonomous_memory 不主动调用 loop.py

**③ hook.py ↔ 所有模块**：
- hook.py 是纯事件总线，不持有任何模块引用
- 其他模块通过 `HookChain.register()` 注册回调
- 事件类型定义在 `ai/types.py`，HookChain 只依赖 types

**④ runtime.py ↔ tool_registry.py**：
- runtime 从 tool_registry 获取父 Agent 的工具集
- runtime._filter_tools() 生产子 Agent 的受限工具集
- 过滤后的工具集传给子 Agent 的 AgentLoop

**⑤ autonomous_memory.py ↔ self_modify.py**：
- crystallize() 调用 quality_score() 评估技能质量
- self_modify 是纯函数，不持有状态

**⑥ prompt.py ↔ capability.py + autonomous_memory.py**：
- prompt.py 从 capability 获取 CAPABILITY_OVERVIEW 片段
- prompt.py 从 autonomous_memory 获取 WORKING_CHECKPOINT + NUDGE 片段
- 片段按 priority 排序组装

### 7.3 生命周期事件时序

```
会话开始：LOOP_START
  └─ 每轮：TURN_START → LLM_CALL → [TOOL_START → TOOL_END]* → TURN_END
      └─ TURN_END 中：
          ├─ update_working_checkpoint()
          ├─ get_nudge_content()
          └─ 如有委托：SUBAGENT_START → [子Agent完整循环] → SUBAGENT_STOP
  └─ 任务完成后：
      ├─ should_crystallize()
      ├─ crystallize() → quality_score()
      └─ start_long_term_update()
会话结束：LOOP_END
```

### 9.2 API 重试策略

```python
# 指数退避 + 最大重试
async def _retry_call(self, fn, max_retries=3, base_delay=1.0):
    for attempt in range(max_retries):
        try:
            return await fn()
        except RateLimitError as e:
            if attempt < max_retries - 1:
                delay = base_delay * (2 ** attempt)
                await asyncio.sleep(delay)
            else:
                raise
        except (ConnectionError, TimeoutError) as e:
            if attempt < max_retries - 1:
                await asyncio.sleep(base_delay)
            else:
                raise
```

### 9.3 Hook 异常隔离

```python
# 核心原则：一个 hook 坏了不能拖垮循环
async def fire(self, event, chain_result=False):
    for handler in self._handlers.get(event.type, []):
        try:
            await handler.callback(event)
        except Exception as e:
            logger.warning(f"Hook '{handler.name}' failed: {e}")
            if chain_result:
                raise  # 极少情况：需要阻断
```

### 9.4 Prompt Token 预算

```python
# 低优先级片段先被裁剪
def build(self, context, max_tokens=8000):
    fragments = sorted(self._fragments.values(), key=lambda f: f.priority)
    parts, token_count = [], 0
    for f in fragments:
        frag_tokens = len(f.content) // 3
        if f.priority <= 24:  # 核心片段不可裁剪
            parts.append(f.content)
            token_count += frag_tokens
        elif token_count + frag_tokens <= max_tokens:
            parts.append(f.content)
            token_count += frag_tokens
    return "\n\n".join(parts)
```
