# Wiki 记忆系统更新（2026-05-04）

## 总览

为 controllable-agent 实现了新的记忆系统，从"手动 API 调用"升级为"Agent 自动管理"。设计灵感来自三个来源：

1. **OpenCode** 的 pointer/mapping 机制（域目录只存指针）
2. **Synthius-Mem 论文** 的抗幻觉结构化记忆（空=没说过=不编造）
3. **Karpathy Wiki** 的层级知识管理（digest 积累后合并为 wiki）

### 存储格式

每条记忆是一个 Markdown 文件，YAML frontmatter 存元数据：

```markdown
---
id: d_015
level: digest
source_session: ses_abc
task_summary: 修复打包问题
tags: [python, 打包]
domains: [conversation]
facts:
  - 用 poetry 打包
  - 添加了 pyproject.toml
confidence: 0.9
---
## Python 打包修复

用户使用 poetry 进行打包管理，修改了 pyproject.toml 中的包发现配置...
```

### 四域映射

域目录的 `_index.md` 只存指针（路径引用），不存内容：

| 域 | 存什么 | 举例 |
|---|--------|------|
| `conversation` | 对话中的关键交互 | `→ ../digest/d_001.md` "Python 打包修复" |
| `profile` | 用户画像（偏好、技能、环境） | `→ ../wiki/dev_environment.md` "开发环境全貌" |
| `agent_view` | Agent 自己的观察和评估 | `→ ../digest/d_012.md` "用户不喜欢 Django" |
| `task` | 任务状态追踪 | `→ ../digest/d_008.md` "进行中：API 设计" |

### 全局倒排索引

`index.md` 维护关键词到文件的倒排映射：

```markdown
## python
- wiki: [Python技术栈](wiki/python_stack.md)
- digest: [d_001](digest/d_001.md), [d_005](digest/d_005.md)
```

### 检索优先级

```
查询 → wiki（最完整）→ digest（更细粒度）→ 空=没说过=不编造
```

### 核心流程

```
AgentLoop.run() 对话结束
    │
    ▼ LOOP_END 携带 AgentResult
    │
    ▼ MemoryExtractor.check_and_extract()
    │
    ├─ TaskDetector.detect()  ← 纯规则，不调 LLM
    │   优先级：continue > topic_switch > done > short_reply > default
    │
    │  任务完成（confidence ≥ 0.6）？
    │   ├─ 否 → 跳过
    │   ▼ 是
    │
    ▼ LLM 提取 digest（固定 prompt，KV Cache 友好）
    │   输入：对话历史
    │   输出：{task_summary, domains, tags, facts, body}
    │
    ├─ FactStore.create_digest() → 写入 digest/d_xxx.md
    ├─ DomainIndex.update_index_for() → 更新倒排索引
    │
    ├─ 同主题 digest ≥ 5？
    │   └─ 是 → LLM 合并为 wiki/xxx.md
    │
    └─ Deduplicator 去重检查
        new → 写入 / duplicate → 跳过 / conflict → 更新 / supplement → 合并
```

### TaskDetector 规则

中文+英文双语正则匹配，四种信号优先级递减：

| 优先级 | 信号类型 | 匹配词 | 结果 |
|--------|---------|--------|------|
| 1（最高） | 继续信号 | 继续/还有/不对/错了/再试/改一下/重写/详细点 | 未完成 |
| 2 | 意图切换 | 接下来/换个话题/另外/顺便/现在帮我 | 完成（旧任务结束） |
| 3 | 完成确认 | 好的/谢谢/完美/没问题/可以了/done/ok | 完成 |
| 4 | 短回复 | ≤10字 + 无工具调用 | 可能完成（confidence 0.6） |

### KV Cache 优化

提取 prompt 分两个固定模板，永远不变：

- `_DIGEST_SYSTEM_PROMPT` — digest 提取指令
- `_WIKI_MERGE_SYSTEM_PROMPT` — wiki 合并指令

首次调用后每次命中缓存，预估单次提取 ≈ 2000 input tokens（80% 缓存命中）+ 500 output tokens。

### Deduplicator 四种判断

| 判断 | 含义 | 处理 |
|------|------|------|
| `new` | 与已有知识无关 | 直接写入 |
| `duplicate` | 已完全包含 | 跳过 |
| `conflict` | 与已有知识矛盾 | merged_body 中给出更新后内容 |
| `supplement` | 补充信息 | merged_body 中给出合并后内容 |
