## 三层架构

```
ai/        ← 零依赖核心类型（Message, Tool, Context, AgentEvent, IModelProvider）
agent/     ← 核心逻辑（AgentLoop, HookChain, MemoryStore, MemoryExtractor...）
app/       ← 入口适配（CLI, TUI, 14工具, 2 Provider, 配置）
```

依赖方向严格：`app/ → agent/ → ai/`，不反向依赖。

## 记忆系统（两套并行）

### 旧系统：L0-L4 层级

```
.agent-memory/
├── index.md                    ← L0 全局索引
└── {project}/
    ├── l1_navigation/          ← L1 场景导航
    ├── l2_facts/               ← L2 环境事实
    ├── l3_experience/          ← L3 任务经验
    ├── l4_sessions/            ← L4 会话归档
    └── agents/{id}/            ← 工作检查点 + 技能卡 + 子Agent结果
```

组件：`MemoryStore`（文件IO）→ `FileSystemMemoryBackend`（L0-L4 CRUD）→ `MemoryIndex`（jieba 搜索）

### 新系统：digest/wiki + 四域（Phase 1+2）

```
.agent-memory/
├── digest/d_001.md             ← 任务摘要（YAML frontmatter + Markdown）
├── wiki/python_stack.md        ← 知识页（同主题 digest ≥5 合并）
├── domains/
│   ├── conversation/_index.md  ← 对话记录指针
│   ├── profile/_index.md       ← 用户档案指针
│   ├── agent_view/_index.md    ← Agent 视角指针
│   └── task/_index.md          ← 任务追踪指针
└── index.md                    ← 全局关键词倒排索引
```

组件：`FactStore`（CRUD）→ `DomainIndex`（四域+倒排索引）→ `TaskDetector`（任务完成判断）→ `MemoryExtractor`（LLM 提取+合并）→ `Deduplicator`（去重/冲突）

### 新记忆自动提取流程

```
用户对话结束 → TaskDetector 启发式判断任务完成
    → 完成(confidence≥0.6) → LLM 提取 digest（固定 prompt，KV Cache 友好）
        → FactStore 写入 → DomainIndex 更新索引
        → 同主题 digest≥5？→ LLM 合并为 wiki
    → 未完成 → 跳过
```

### 检索优先级（抗幻觉）

```
查询 → wiki（最完整）→ digest（更细粒度）→ 空=没说过=不编造
```

## 上下文压缩（三层递进）

| 层 | 方法 | 调 LLM？ | 做什么 |
|---|------|---------|--------|
| Layer 1 Prune | 清除 3 轮前的工具输出 | 否 | 替换为 `[Old tool result content cleared]` |
| Layer 2 Summary | 压缩旧消息为结构化摘要 | 模板化 | 目标/指示/发现/完成情况/相关文件 |
| Layer 3 Emergency | 从最早消息逐条删除 | 否 | 直到 token 降到 98% 以下 |

触发时机：AgentLoop 每次调 LLM 前检测，超过 85% 阈值触发。
