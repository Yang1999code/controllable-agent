## 更新了什么？（大白话版）

### 第一波：修基础 bug（4 个）

| 问题 | 大白话 | 怎么修的 |
|------|--------|---------|
| 启动报错 | 每次启动都蹦出一个黄色警告 | 改了一行代码，直接 await 就好了 |
| 中文乱码 | one-shot 模式下终端输出全是乱码 | 强制终端用 UTF-8 编码 |
| 文件写错位置 | AI 不知道自己在哪个目录，文件到处乱写 | 在提示词里告诉它当前工作目录 |
| 交互式程序卡死 | 让 AI 跑一个有 input() 的程序，它就卡在那了 | 告诉 AI 别跑需要手动输入的程序 |

**效果**: 基本功能终于正常了，不会启动报错、中文乱码、文件乱放。

---

### 第二波：修多智能体协作 bug（2 个）

| 问题 | 大白话 | 怎么修的 |
|------|--------|---------|
| 子 Agent 干到一半被强制停止 | AI 写代码写到一半被"最多调 10 次工具"限制掐断了 | 给不同角色设了不同的工具调用上限（写代码的给 50 次，查代码的给 30 次） |
| Windows 上输出 emoji 崩溃 | 子 Agent 输出带 ✅ 这样的表情符号，在 Windows 上直接报错 | 对输出做了一次安全编码处理 |

**效果**: 多智能体协作终于能完整跑完任务了，不再半途而废或崩溃。

---

### 第三波：对抗审查 + 角色强化（3 个改进）

| 功能 | 大白话 | 效果 |
|------|--------|------|
| 红队审查 | 派一个"找茬的"AI，专门故意找代码里的 bug | 代码质量明显提高，能发现类型注解 bug、安全隐患等 |
| Reviewer 检查清单 | 给审查员一份具体要查什么的清单 | 审查员从"走形式"变成"真干活"了 |
| 工具调用上限调整 | 写代码的 AI 从 30 次提到 50 次工具调用 | 大项目（比如 VS Code 级别）也能完整写完 |

**效果**: 生成的代码质量从"能用"提升到"比较靠谱"，安全漏洞和类型错误都能被发现。

### 第四波：三项核心优化

| 功能 | 大白话 | 效果 |
|------|--------|------|
| 记忆自动提取 | 多智能体协作完成后，自动总结经验存到记忆库 | 下次遇到类似任务可以复用经验 |
| 任务完成检测增强 | AI 能感知"我用了多少次工具"、"干了几轮了"，更准确判断任务是否完成 | 不会提前结束也不会无限拖延 |
| 修复-验证循环 | 发现严重 bug 后自动修复 → 再验证 → 还有 bug 就再修（最多 2 轮） | 对抗审查发现的 bug 能自动修好，不用人工干预 |

**效果**: Agent 有了"自我纠错"能力，发现问题能自己修好再验证。

### 第五波：两项功能增强

| 功能 | 大白话 | 效果 |
|------|--------|------|
| 工具调用上限动态调整 | AI 会先看任务有多复杂，简单任务少给点资源，复杂任务多给点 | 简单任务不浪费资源，复杂任务不会做不完 |
| 技能结晶持久化 | AI 完成任务后，会把可复用的经验自动保存成"技能卡"，下次直接用 | 做过的任务类型会越做越好，技能可以积累 |

**动态调整具体分三档**:
- 简单任务（3 个文件以内）: coder 最多调 30 次工具
- 中等任务（4-8 个文件）: coder 最多调 50 次
- 复杂任务（9+ 个文件，10+ 步骤）: coder 最多调 80 次

**技能卡示例**:
```yaml
name: create_flask_api
description: 创建 Flask REST API 项目
trigger_condition: flask api rest 创建
steps:
  - tool: write
    description: 创建主文件
  - tool: bash
    description: 运行测试
```
下次你说"帮我创建一个 Flask API"，AI 会自动找到这个技能卡复用。

### 第六波：五项进阶优化（全部完成 ✅）

| 功能 | 大白话 | 效果 |
|------|--------|------|
| 子 Agent 串行化 | 以前 coder 和 reviewer 同时开工，reviewer 看不到 coder 的最终代码。现在 coder 先写完，reviewer 再审查 | 代码审查质量大幅提升，reviewer 能看到完整代码 |
| 技能质量评分 | 每次使用技能后根据成功/失败更新分数。用得越多，单次影响越小。低分技能自动淘汰 | 技能库越用越精准，烂技能自动消失 |
| 动态超时 | 简单任务给 5 分钟，中等任务给 10 分钟，大项目给 15 分钟 | 不再出现"简单任务超时浪费"或"大项目做不完"的问题 |
| 上下文压缩增强 | 新增第 0 层：自动截断超大工具输出（>5000 字符），保留头尾关键信息 | 大项目运行时不会因为工具输出太大而爆内存 |
| MCP 自动发现 | 放一个 YAML 文件到 `.agent-base/mcp/` 目录，启动时自动连接注册 | 不用改配置文件就能加新的 MCP 工具服务器 |

**技能评分规则**:
- 成功使用: +10 分
- 失败使用: -20 分
- 用户反馈: 0-100 分加权
- 使用≥3次且分数<30: 自动淘汰

**动态超时分档**:
- 简单（≤3 文件）: coder 5 分钟, reviewer 5 分钟
- 中等（4-8 文件）: coder 10 分钟, reviewer 7 分钟
- 复杂（≥9 文件）: coder 15 分钟, reviewer 10 分钟

## 技术细节（给开发者看的）

### 多智能体编排流程（最新版）

```
用户请求
  ↓
Phase A:  Planner（分析需求，写计划）
  ↓
Phase A+: 分析计划复杂度 → 动态调整资源和超时
  ↓
Phase B-1: Coordinator + Memorizer（并行，轻量辅助）
  ↓
Phase B-2: Coder（串行，先写完代码）
  ↓
Phase B-3: Reviewer（串行，等 coder 写完再审查）
  ↓
Phase C:  Reviewer 集成测试（跑 pytest）
  ↓         ↓ 失败 → 回 Phase B（最多 3 次）
Phase C+: 红队审查（故意找 bug）
  ↓
Phase Fix:   Coder 修复问题 ← 有 CRITICAL/HIGH 问题才进
Phase Verify: Reviewer 验证修复 ← 最多 2 轮
  ↓
Phase D:  Memorizer 总结 + 技能结晶
  ↓
Phase Memory: 统一记忆提取（写入 digest）
  ↓
Phase Skill: 技能持久化（写入 YAML）
  ↓
返回结果
```
