## 2026-04-30 — MCP (Model Context Protocol) 支持

### 背景

MCP 是 Anthropic 推出的开放协议，标准化了 LLM 与外部工具/资源的连接方式。接入 MCP 生态后，my-agent 可以直接使用数千个 MCP Server（文件系统、数据库、API 服务等），不需要逐个写适配代码。

### 新增文件

| 文件 | 说明 |
|------|------|
| `agent/mcp/__init__.py` | 公共导出（`MCPClient`, `MCPServerConfig`, `MCPToolAdapter`） |
| `agent/mcp/client.py` | MCP Client 完整实现（~190 行） |
| `tests/agent/test_mcp.py` | 16 个单元测试 |

### 设计要点

**MCPClient** — 单个 MCP Server 的持久客户端：
- 管理传输生命周期（stdio 子进程 / SSE 长连接）
- 连接时自动发现工具（`session.list_tools()`）
- 工具调用串行化（`asyncio.Lock`，MCP session 不支持并发）

**MCPToolAdapter** — 将 MCP 工具包装为 `ITool` Protocol 兼容对象：
- `inputSchema` (JSON Schema) → `list[ToolParameter]` 自动转换
- 鸭子类型，无需显式继承 ITool，直接注册到 ToolRegistry

**MCPServerConfig** — 连接配置 dataclass：
- `transport`: `"stdio"`（启动子进程）或 `"sse"`（HTTP SSE 端点）
- `disabled: bool` 跳过某个服务器

### CLI 集成

在 YAML 配置中声明 MCP 服务器：

```yaml
mcp_servers:
  - name: "filesystem"
    transport: "stdio"
    command: "npx"
    args: ["-y", "@modelcontextprotocol/server-filesystem", "/data"]
  - name: "github"
    transport: "sse"
    url: "https://api.github.com/mcp"
    disabled: true   # 暂时跳过
```

CLI 启动时自动连接、注册工具；退出时 `try/finally` 确保断开。

### 验证结果

- ✅ 16 个新增 MCP 测试通过
- ✅ 全部 222 个测试通过
- ✅ `mcp` 包为可选依赖（未安装时优雅跳过，不报错）
- ✅ `from my_agent import MCPClient, MCPServerConfig, MCPToolAdapter` 正常

## 2026-04-30 — 前端解耦 + 公共 API 规范化

### 背景

参照 OpenAI Agents SDK 的规范化设计模式（`from agents import Agent, Runner`），对 my-agent 进行轻量级解耦和规范化改进。核心原则：**不改功能、不改性能、只改 import 路径和接入方式**。

### 改动概览

| 文件 | 改动类型 | 说明 |
|------|----------|------|
| `my_agent.py` | **新增** | 公共 API 统一入口，集中重导出 |
| `ai/types.py` | 增加 | 新增 `IUiSession` Protocol |
| `agent/loop.py` | 增加 | 新增 `model_name`、`tool_count` 公共属性 |
| `app/tui/session.py` | 重构 | 斜杠命令注册表化 + 移除内部渗透 |
| `app/cli.py` | 重构 | 改用公共入口 import + IUiSession 类型标注 |

### 详细变更

#### 1. 新建 `my_agent.py` — 公共 API 入口

用户代码现在可以一行导入：

```python
from my_agent import AgentLoop, Context, ToolRegistry, FlowInspector
```

不再需要深入内部模块路径 `from agent.loop import AgentLoop`。文件集中重导出 20+ 核心类型，参考 OpenAI SDK `src/agents/__init__.py`。

**好处**：内部重构不影响外部使用；IDE 自动完成只需记住一个入口。

#### 2. `ai/types.py` — 新增 IUiSession Protocol

```python
@runtime_checkable
class IUiSession(Protocol):
    async def run(self) -> None: ...
```

CLI 通过这个 Protocol 引用 TUI 会话，不再直接依赖 `TuiSession` 具体类。以后替换为 Web/API 前端时，只需实现 `IUiSession` 即可，CLI 一行不改。

**好处**：前端可替换性；Protocol 零运行时开销。

#### 3. `agent/loop.py` — 暴露公共属性

新增两个 `@property`：

- `model_name` → `getattr(self.provider, "model", "unknown")`
- `tool_count` → `len(self.tools.tools)`

前端不再穿透 `loop.provider.model` 和 `loop.tools.tools` 获取信息。私有属性改名不影响前端。

**好处**：封装内部结构；前端与 provider/tools 内部实现解耦。

#### 4. `app/tui/session.py` — 斜杠命令注册表化

**之前**：`_handle_command()` 是一个 50 行的 if-elif 链，硬编码 6 个命令在方法内部。每加一个命令就改一次类。

**之后**：
- `COMMANDS: dict[str, tuple[str, Callable]]` 全局注册表
- `register_command(name, description)` 装饰器
- `_handle_command()` 变成 15 行的查表 + 别名解析
- 7 个内置命令各是一个独立的模块级 async 函数

```python
@register_command("/help", "显示帮助")
async def _cmd_help(session: TuiSession): ...
```

**好处**：插件可通过 `register_command` 装饰器扩展命令；命令逻辑独立可测；别名映射集中管理。

#### 5. `app/cli.py` — 改用公共入口

- **import 改为** `from my_agent import Context, Message, ToolRegistry, ...`
- **`run_legacy_repl()`** 改用 `loop.model_name` 和 `loop.tool_count`（不再穿透 `loop.provider.model`）
- **session 变量**加上 `IUiSession` 类型标注

**好处**：CLI 成为 my_agent 公共 API 的"用户"，验证了公共 API 的可用性。

### 验证结果

- ✅ 全部 206 个测试通过
- ✅ `from my_agent import *` 所有导出正常
- ✅ `IUiSession` Protocol 可被 `TuiSession` 结构匹配
- ✅ 所有斜杠命令功能不变（`/help`、`/flowchart`、`/fco`、`/clear`、`/model`、`/tools`、`/tokens`）

### 前后对比

| 维度 | 之前 | 之后 |
|------|------|------|
| 用户入口 | 7 条不同路径的 import | `from my_agent import ...` 单入口 |
| TUI 依赖 | 耦合 `TuiSession` 具体类 | 依赖 `IUiSession` Protocol |
| 命令扩展 | 改 if-elif 链 | 装饰器注册 |
| 前端穿透 | `loop.provider.model` | `loop.model_name` property |
| 模块引用 | 直接引用 `agent/` 内部路径 | 通过公共 API 间接引用 |
