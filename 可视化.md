# 终端可视化设计文档

> 对标 Claude Code / OpenCode 的终端 UI 设计，为 controllable-agent 提供可落地的改造方案。

---

## 一、现状分析

### 当前实现（app/tui/）

| 模块 | 行数 | 能力 | 短板 |
|------|------|------|------|
| `display.py` | 194 | ANSI 颜色、`\|/-\\` 旋转 spinner、`[*]` 工具前缀、状态行 | 无流式输出、无 Markdown 渲染、无思维链展示 |
| `session.py` | 298 | AgentLoop → 终端桥接、prompt_toolkit scrollback | 无权限提示、无会话管理 |
| `input_area.py` | 128 | 多行输入、Ctrl+Enter 提交 | 无 @ 引用、无 ! 命令、无历史补全 |
| `flowchart.py` | 799 | 双层交互流程图 | 仅流程图，非主交互界面 |

### 与标杆的差距

| 能力 | Claude Code | OpenCode | 当前状态 |
|------|-------------|----------|----------|
| 流式输出 | 60 FPS React+Ink 实时渲染 | 实时 streaming | ❌ 等完整响应后才显示 |
| Markdown 渲染 | 语法高亮 + diff 着色 | 主题化渲染 | ❌ 纯文本输出 |
| 思维链展示 | 可展开的 thinking 区块 | `/thinking` 切换显示 | ❌ 完全隐藏 |
| 工具调用展示 | 彩色徽章 + 动词标签 + 进度 | `/details` 展开/收起 | ⚠️ 简单 `[*]` 前缀 |
| 权限提示 | 内联确认框 + Allow/Deny | Y/N 确认 | ❌ 无 |
| 多 Agent 展示 | Tab 切换 Agent | Tab 切换 + 状态栏 | ❌ 单一视图 |
| 会话管理 | /clear, /compact | 会话列表 + 恢复 | ⚠️ 仅 exit |
| 主题 | 自适应终端 | 多主题 + 自定义 | ⚠️ 固定配色 |
| Diff 可视化 | 语法高亮 diff | 内联 diff | ❌ 无 |

## 二、设计目标

**核心原则：信息密度 > 视觉花哨。** 终端空间有限，每一行都要有信息增量。

1. **流式优先**：用户看到的是逐字出现的响应，不是等 10 秒后的完整输出
2. **结构清晰**：思维链、工具调用、文本回复三个层次视觉分离
3. **操作直觉**：常用操作有快捷键，无需记忆命令
4. **兼容性强**：Windows Terminal + PowerShell / cmd / Git Bash 均可用

```

│ - old_code
│ + new_code
│
│ ? 允许编辑 src/sort.py？ [Y]es / [N]o / [A]lways
```

- 默认选项高亮（Y）
- 支持键盘快捷键（y/n/a）

## 五、流式输出设计

### 5.1 流式架构

```
LLM API (SSE Stream)
    │
    ▼
TokenReceiver  ──── 接收 SSE 事件
    │
    ▼
StreamBuffer   ──── 缓冲 + 解析 Markdown 结构
    │
    ▼
TermRenderer   ──── 增量渲染到终端
```

### 5.2 增量渲染策略

| 内容类型 | 渲染方式 | 更新频率 |
|----------|----------|----------|
| 纯文本 | 逐字追加 | 每个 token |
| 代码块 | 缓冲到 ` ``` ` 闭合后整体渲染 | 块级 |
| 列表 | 逐项渲染 | 每个 `\n` |
| 标题 | 整行渲染 | 行级 |
| 工具调用 JSON | 解析后渲染为结构化视图 | 一次性 |

### 5.3 Markdown 终端渲染规则

```python
# 标题
"# 标题" → "\033[1;4m标题\033[0m"          # 粗体+下划线
"## 标题" → "\033[1m标题\033[0m"             # 粗体
"### 标题" → "\033[1m── 标题 ──\033[0m"     # 粗体+装饰

# 代码块
"```python\ncode\n```" → 带左边框的绿色代码区

# 行内代码
"`code`" → "\033[7mcode\033[0m"              # 反色高亮

# 列表
"- item" → "  • item"
"1. item" → "  1. item"

# 粗体/斜体
"**bold**" → "\033[1mbold\033[0m"
"*italic*" → "\033[3mitalic\033[0m"

# 链接
"[text](url)" → "\033[4mtext\033[0m \033[2m(url)\033[0m"

# 表格
保持等宽对齐，使用 box-drawing 字符
```

### 5.4 代码块渲染

```
  ┌─ python ──────────────────────────────┐
  │ def quicksort(arr):                    │
  │     if len(arr) <= 1:                  │
  │         return arr                     │
  │     pivot = arr[len(arr) // 2]         │
  │     left = [x for x in arr if x < pivot]│
  │     right = [x for x in arr if x > pivot]│
  │     return quicksort(left) + [pivot] +  │
  │            quicksort(right)             │
  └────────────────────────────────────────┘
```

## 七、实现路线

### Phase 1：基础增强（P0，~3 天）

改造 `display.py` 和 `session.py`：

1. **流式输出支持**
   - 修改 `AgentLoop` 添加 `on_token` 回调
   - `StreamBuffer` 类缓冲 + 解析 token
   - `TermRenderer` 增量渲染

2. **工具调用视觉升级**
   - 替换 `[*]` 为图标 + 动词标签
   - 折叠/展开模式
   - 执行结果着色

3. **状态栏增强**
   - 显示模型名、turn 数、token 用量
   - 实时更新

### Phase 2：Markdown + 思维链（P1，~3 天）

1. **Markdown 渲染器**
   - `MarkdownTermRenderer` 类
   - 标题、代码块、列表、粗体、斜体、链接
   - 代码块带语言标签和边框

2. **思维链展示**
   - 检测 `<thinking>` 标签
   - 灰色折叠区块
   - `Ctrl+T` 切换显示

3. **权限提示**
   - 文件写入/编辑前确认
   - Shell 命令执行前确认
   - 简洁 diff 预览

### Phase 3：多 Agent + 高级交互（P2，~4 天）

1. **多 Agent 状态栏**
   - Agent 状态图标
   - Tab 切换视图
   - 编排进度条

2. **快捷键系统**
   - 完整快捷键映射
   - Tab 补全
   - 历史命令

3. **主题系统**
   - 3 套预置主题（默认/深色/高对比）
   - YAML 配置自定义

## 八、技术约束

### 8.1 兼容性矩阵

| 终端 | Unicode | ANSI 256色 | Alt-screen | 备注 |
|------|---------|-----------|------------|------|
| Windows Terminal | ✓ | ✓ | ✓ | 推荐使用 |
| PowerShell 7+ | ✓ | ✓ | ✗ | 大部分功能可用 |
| Git Bash | ✓ | ✓ | ✗ | 大部分功能可用 |
| CMD | ✗ | ✗ | ✗ | 兼容模式 |
| VSCode 终端 | ✓ | ✓ | ✗ | 大部分功能可用 |

### 8.2 性能预算

| 指标 | 目标 | 说明 |
|------|------|------|
| 首字延迟 | < 100ms | 从收到 token 到屏幕显示 |
| 渲染帧率 | ≥ 30 FPS | prompt_toolkit 刷新率 |
| 内存增量 | < 50MB | 流式缓冲区上限 |
| CPU 占用 | < 5% | 空闲时 |

### 8.3 依赖选择

| 依赖 | 用途 | 是否已有 | 备注 |
|------|------|----------|------|
| `prompt_toolkit` | 输入 + 布局 | ✓ | 已在用 |
| `rich` | Markdown 渲染 + 表格 | ✗ | 推荐新增 |
| `textual` | 全屏 TUI 框架 | ✗ | 可选，重量级 |

**推荐方案：prompt_toolkit + rich**

理由：
- prompt_toolkit 已在用，迁移成本低
- rich 提供开箱即用的 Markdown 渲染、表格、进度条
- 组合轻量（rich ~2MB），满足需求
- 无需引入 textual 的重量级框架

### 关键类设计

```python
# stream.py
class StreamBuffer:
    """流式 token 缓冲，解析 Markdown 结构边界。"""
    def feed(self, token: str) -> list[RenderChunk]: ...
    def flush(self) -> list[RenderChunk]: ...

class TermRenderer:
    """增量渲染到终端，最小化重绘。"""
    def render(self, chunks: list[RenderChunk]) -> None: ...
    def clear_line(self) -> None: ...

# markdown.py
class MarkdownTermRenderer:
    """Markdown → ANSI 彩色文本。"""
    def render(self, md: str) -> str: ...
    def render_inline(self, md: str) -> str: ...
    def render_code_block(self, code: str, lang: str) -> str: ...

# theme.py
class Theme:
    """配色主题。"""
    name: str
    colors: dict[str, str]  # 元素名 → ANSI 颜色

DEFAULT_THEME = Theme(...)
DARK_THEME = Theme(...)
HIGH_CONTRAST_THEME = Theme(...)

# widgets.py
class StatusBar:
    """顶部状态栏。"""
    def render(self, model, turn, tokens, agents) -> str: ...

class PermissionPrompt:
    """权限确认框。"""
    def render(self, tool, args, diff) -> str: ...

class AgentProgressBar:
    """多 Agent 编排进度。"""
    def render(self, phases) -> str: ...

class ToolCallBadge:
    """工具调用徽章。"""
    def render(self, tool, args, status) -> str: ...
    def render_expanded(self, tool, args, result) -> str: ...
```
